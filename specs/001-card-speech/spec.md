# Feature Specification: Card Text-to-Speech

**Feature Branch**: `001-card-speech`
**Created**: 2026-03-04
**Status**: Draft
**Input**: User description: "カードの読み上げ機能をつけてください。Web Speech APIで良いです。"

## User Scenarios & Testing _(mandatory)_

### User Story 1 - カードを手動で読み上げる (Priority: P1)

復習セッション中、ユーザーはカードに表示されているテキスト（表面・裏面それぞれ）を音声で聞くことができる。読み上げボタンをタップするだけで、その場でカード内容が読み上げられる。

**Why this priority**: 読み上げ機能の最小限のコア価値を届ける MVP。ボタン1つで動作し、他のユーザーストーリーの基盤となる。

**Independent Test**: 復習画面を開き、カード表面の読み上げボタンをタップする。音声が再生されれば成功。単独でデプロイ・デモ可能。

**Acceptance Scenarios**:

1. **Given** 復習中にカード表面が表示されている、**When** ユーザーが読み上げボタンをタップする、**Then** カード表面のテキストが音声で読み上げられる
2. **Given** カードを裏返して答えが表示されている、**When** ユーザーが読み上げボタンをタップする、**Then** カード裏面のテキストが音声で読み上げられる
3. **Given** 音声が再生中である、**When** ユーザーが再度読み上げボタンをタップする、**Then** 再生が停止される（トグル動作）
4. **Given** 音声が再生中である、**When** ユーザーが次のカードへ進む、**Then** 再生中の音声が自動停止される

---

### User Story 2 - カード表示時に自動読み上げ (Priority: P2)

ユーザーが設定で自動読み上げを有効にすると、カードが表示されるたびに自動的にテキストが読み上げられる。ながら学習や手を使えない状況での利用を想定。読み上げ完了後のカードフリップは自動では行われず、ユーザーが手動でめくったタイミングで裏面の読み上げが始まる。

**Why this priority**: P1 の手動読み上げ基盤の上に構築でき、ハンズフリー学習という付加価値を提供する。

**Independent Test**: 設定で自動読み上げをオンにして復習を開始する。各カードが表示されるたびに自動再生されれば成功。

**Acceptance Scenarios**:

1. **Given** 自動読み上げが有効で復習中、**When** 新しいカードが表示される、**Then** カード表面のテキストが自動的に読み上げられる
2. **Given** 自動読み上げが有効で表面の読み上げが完了した、**When** そのまま放置する、**Then** カードは自動でめくれず、ユーザーの操作を待つ
3. **Given** 自動読み上げが有効でカードを裏返した直後、**When** 裏面が表示される、**Then** カード裏面のテキストが自動的に読み上げられる
4. **Given** 自動読み上げが有効で再生中、**When** ユーザーが手動で読み上げを停止する、**Then** 現在のカードの再生のみ停止され、次のカードでは自動読み上げが継続される
5. **Given** 自動読み上げが有効、**When** ユーザーが設定でオフにする、**Then** 以降のカードでは自動読み上げが行われない

---

### User Story 3 - 読み上げ速度の調整 (Priority: P3)

ユーザーは読み上げ速度（遅め・標準・速め）を設定できる。語学学習など聞き取りの難易度調整に役立てる。

**Why this priority**: 核心機能ではないが、学習効果と快適性を高める。P1・P2 が完成後に追加できる拡張機能。

**Independent Test**: 設定で「遅め」を選択し読み上げをタップ。通常より遅い速度で読み上げられれば成功。

**Acceptance Scenarios**:

1. **Given** 設定画面で速度を「遅め」に設定した、**When** 読み上げボタンをタップする、**Then** 標準より遅い速度でテキストが読み上げられる
2. **Given** 設定画面で速度を「速め」に設定した、**When** 読み上げボタンをタップする、**Then** 標準より速い速度でテキストが読み上げられる

---

### Edge Cases

- カードのテキストが空の場合、読み上げボタンは非活性状態で表示され、タップしても何も起きない
- デバイスやブラウザが音声合成に対応していない場合、読み上げボタンを非表示にし、他の機能は通常通り動作する
- 読み上げ中にアプリがバックグラウンドに移動した場合、再生を停止する
- カードに日本語と英語が混在している場合、テキスト全体をそのまま読み上げる（言語ごとに分割しない）
- 非常に長いテキスト（500文字超）でも読み上げが途中で途切れずに完了する
- 自動読み上げが有効でも表面の読み上げ完了後にカードは自動でめくれない。読み上げ完了とカードフリップは独立した操作である

## Requirements _(mandatory)_

### Functional Requirements

- **FR-001**: ユーザーは復習画面のカード表面・裏面それぞれに対して読み上げを個別に開始できる
- **FR-002**: ユーザーは読み上げをいつでも任意のタイミングで停止できる
- **FR-003**: 読み上げ中に次のカードへ進むと、自動的に再生が停止される
- **FR-004**: ユーザーは設定画面で自動読み上げのオン/オフを切り替えられる
- **FR-005**: ユーザーは設定画面で読み上げ速度（遅め・標準・速め の3段階）を選択できる
- **FR-006**: 読み上げ設定は認証済みユーザーIDに紐付けてローカルストレージに保存され、次回起動後も維持される
- **FR-007**: 音声合成が利用できない環境では、読み上げ機能が非表示になり、他の機能への影響がない
- **FR-008**: テキストが空のカードでは読み上げボタンが操作できない状態で表示される

### Assumptions

- 読み上げの言語はカードのテキストに基づいてブラウザが自動判定するものとし、ユーザーが言語を個別に指定する機能は含まない
- 設定は認証済みユーザーIDをキーとしてフロントエンドのローカルストレージに保存する（サーバーサイド同期は対象外）。複数ユーザーが同一デバイスを使用した場合でも、各ユーザーの設定は独立して管理される
- 読み上げ対象はカードのテキストフィールドのみとし、デッキ名やメタ情報は含まない

## Success Criteria _(mandatory)_

### Measurable Outcomes

- **SC-001**: ユーザーが読み上げボタンをタップしてから音声再生が開始されるまで1秒以内
- **SC-002**: 音声合成をサポートする主要ブラウザ（Chrome、Safari、Edge、Firefox）のすべてで読み上げが正常に動作する
- **SC-003**: 音声合成非対応環境において、読み上げ機能の欠如を除き通常通り復習を完了できる（エラーゼロ）
- **SC-004**: 自動読み上げを有効にした状態でフルの復習セッション（10枚のカード）を画面を見ずに完了できる

## Clarifications

### Session 2026-03-04

- Q: 自動読み上げ完了後にカードが自動フリップするか、それともユーザーが手動でめくるか？ → A: 自動フリップなし。読み上げ完了後もユーザーが手動でめくり、裏返したタイミングで裏面を自動再生する
- Q: 自動読み上げ中にユーザーが手動停止した場合、次のカードでも自動再生は継続されるか、それとも設定がオフになるか？ → A: 手動停止は現在カードのみに適用され、次のカードでは自動読み上げが継続される
- Q: 設定の保存スコープ — FR-006「ユーザーごとに保存」と Assumptions「ローカルストレージに保存」の矛盾について、同一デバイスを複数ユーザーが使う場合の保存スコープはどちらですか？ → A: 認証済みユーザーIDに紐付けてローカルストレージに保存する
