# Quickstart: Card Text-to-Speech — 手動テストシナリオガイド

**Feature Branch**: `001-card-speech`
**対象**: QA / レビュアー / 開発者が本機能を手動で確認するためのガイド

---

## 前提条件

- ローカル開発環境が起動していること（`cd frontend && npm run dev`）
- 少なくとも 10 枚以上のカードを含むデッキが存在すること
- テストデバイス: デスクトップブラウザ（音声出力付き）

---

## SC-001: 読み上げボタンタップ → 音声開始まで 1 秒以内

**検証項目**: ユーザーが読み上げボタンをタップしてから音声再生が開始されるまで 1 秒以内であること

### 手順

1. 復習セッションを開始する
2. カード表面が表示された状態で、カード上の読み上げボタン（スピーカーアイコン）を確認する
3. スピーカーボタンをタップ（クリック）する
4. ストップウォッチなどで、タップから音声が聞こえるまでの時間を計測する

### 合格基準

- [ ] タップから 1 秒以内に音声再生が開始される
- [ ] 読み上げボタンが再生中状態（アイコン変化や `aria-label` の変化）になる
- [ ] カード表面のテキスト内容が読み上げられる

### 補足確認（US1 シナリオ全体）

| シナリオ | 操作 | 期待動作 |
|---------|------|---------|
| 手動読み上げ（表面） | カード表面のスピーカーボタンをタップ | 表面テキストが読み上げられる |
| 手動読み上げ（裏面） | カードを裏返してスピーカーボタンをタップ | 裏面テキストが読み上げられる |
| 停止（トグル） | 再生中にスピーカーボタンを再タップ | 再生が停止される |
| 次カードへ進む | 再生中に次のカードへ進む | 自動的に再生が停止される |

## SC-002: 主要ブラウザ動作確認（Chrome / Safari / Edge / Firefox）

**検証項目**: 音声合成をサポートする主要ブラウザのすべてで読み上げが正常に動作すること

### テスト対象ブラウザ

| ブラウザ | 最小バージョン | 確認日 | 結果 |
|---------|--------------|--------|------|
| Google Chrome | 最新版 | | ☐ PASS / ☐ FAIL |
| Apple Safari | 最新版 | | ☐ PASS / ☐ FAIL |
| Microsoft Edge | 最新版 | | ☐ PASS / ☐ FAIL |
| Mozilla Firefox | 最新版 | | ☐ PASS / ☐ FAIL |

### 各ブラウザで確認する手順

1. 対象ブラウザでアプリを開く
2. 復習セッションを開始する
3. カード表面の読み上げボタンをタップして音声が再生されることを確認する
4. カードを裏返して裏面の読み上げボタンをタップして音声が再生されることを確認する
5. ブラウザのデベロッパーコンソール（F12）を開き、エラーが出ていないことを確認する

### 合格基準（各ブラウザ共通）

- [ ] 読み上げボタンが表示される
- [ ] タップで音声が再生される
- [ ] 再タップで音声が停止される
- [ ] コンソールエラーが発生しない
- [ ] 設定画面で読み上げ設定セクションが表示される

## SC-003: 音声合成非対応環境でエラーゼロ・通常復習完了

**検証項目**: 音声合成非対応環境において、読み上げ機能の欠如を除き通常通り復習を完了できること（エラーゼロ）

### 方法 A: 古いブラウザ / 非対応環境でテスト

音声合成非対応のブラウザが手元にある場合は、そのブラウザで以下を確認する。

### 方法 B: `speechSynthesis` を無効化してテスト（推奨）

ブラウザのデベロッパーコンソールで以下を実行して、音声非対応環境をシミュレートする:

```javascript
// speechSynthesis を undefined に置き換えてシミュレート
Object.defineProperty(window, 'speechSynthesis', { get: () => undefined });
```

その後、ページをリロードせずに以下を確認する（またはリロード後に確認）。

> **注意**: この設定は当該タブのみに影響し、リロードで元に戻る。

### 手順

1. デベロッパーコンソールで音声合成を無効化する（方法 B）または非対応ブラウザを使用する
2. 復習セッションを開始する
3. カード表面 / 裏面に読み上げボタンが**表示されないこと**を確認する
4. 設定画面を開き、音声設定セクションに「ご利用のブラウザは音声合成に対応していません」等のメッセージが表示されることを確認する
5. 通常の復習操作（カード評価・次のカードへの進行）がすべて正常に動作することを確認する
6. 10 枚のカードを復習し完了画面に到達することを確認する

### 合格基準

- [ ] 読み上げボタンが表示されない（非対応環境では非表示）
- [ ] 設定画面に非対応メッセージが表示される（音声設定セクション非表示）
- [ ] カードのめくり・評価・次カードへの進行が正常に操作できる
- [ ] コンソールに JavaScript エラーが一切出力されない
- [ ] 復習セッションを最後まで完了できる

## SC-004: 自動読み上げ有効 + 10 枚カードのハンズフリーセッション

**検証項目**: 自動読み上げを有効にした状態でフルの復習セッション（10 枚のカード）を画面を見ずに完了できること

### 前提

- 10 枚以上のカードを含むデッキが存在すること
- 各カードの表面・裏面にテキストが入力されていること

### 手順

1. **設定で自動読み上げをオンにする**
   - 設定画面（SettingsPage）を開く
   - 「音声設定」セクションの「自動読み上げ」トグルをオンにする

2. **読み上げ速度を確認する（任意）**
   - 「読み上げ速度」の設定を「遅め」に変更して聞き取りやすくしてもよい

3. **復習セッションを開始する**
   - 10 枚以上含むデッキの復習を開始する

4. **ハンズフリー確認（画面を見ずに操作）**
   - カードが表示される → 表面テキストが自動的に読み上げられることを確認する
   - 読み上げ完了後、カードは自動でめくれないことを確認する（ユーザー操作待ち）
   - カードをめくる → 裏面テキストが自動的に読み上げられることを確認する
   - 評価ボタンを押す → 次のカードに進み、表面テキストが自動的に読み上げられることを確認する
   - これを 10 枚繰り返す

5. **途中で手動停止を試みる（US2 シナリオ確認）**
   - 自動読み上げ中にスピーカーボタンをタップして停止する
   - 次のカードに進んだ際、自動読み上げが再開されることを確認する

### 合格基準

- [ ] 各カード表面が表示されると自動的に読み上げが開始される
- [ ] 表面の読み上げ完了後、カードは自動でめくれない（ユーザーの操作待ち）
- [ ] カードを裏返すと裏面テキストが自動的に読み上げられる
- [ ] 手動停止後、次のカードでは自動読み上げが継続される
- [ ] 10 枚のカードすべての復習を完了できる
- [ ] コンソールエラーが発生しない

### 速度設定の確認（US3）

| 速度設定 | 期待 |
|---------|------|
| 遅め (0.5x) | 標準より明らかに遅いペースで読み上げられる |
| 標準 (1.0x) | 通常の自然な速度で読み上げられる |
| 速め (1.5x) | 標準より明らかに速いペースで読み上げられる |
