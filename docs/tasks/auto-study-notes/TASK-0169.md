## 実装詳細

### 1. POST /study-notes/generate 🔵

**信頼性**: 🔵 *api-endpoints.md より*

リクエストボディの source_type, source_id を受け取り、StudyNotesService.generate_or_get() を呼び出す。

### 2. GET /study-notes 🔵

**信頼性**: 🔵 *api-endpoints.md より*

クエリパラメータの source_type, source_id を受け取り、キャッシュを返却。キャッシュなしの場合は data: null。

### 3. エラーハンドリング 🔵

**信頼性**: 🔵 *既存ハンドラーパターン・api-endpoints.md より*

InsufficientCardsError → 400, AITimeoutError → 504, AIRateLimitError → 429。

### 4. ルーティング追加 🔵

**信頼性**: 🔵 *既存template.yamlパターンより*

template.yaml のEvents セクションに study-notes のルーティングを追加。

## 単体テスト要件

### テストケース1: POST 正常系 🔵

**Given**: 有効なリクエスト
**When**: POST /study-notes/generate
**Then**: 200 OK + StudyNotesResponse

### テストケース2: POST カード数不足 🔵

**Given**: 5枚未満のデッキ
**When**: POST /study-notes/generate
**Then**: 400 INSUFFICIENT_CARDS

### テストケース3: GET キャッシュヒット 🔵

**Given**: キャッシュ済みデッキ
**When**: GET /study-notes
**Then**: 200 OK + キャッシュデータ

### テストケース4: GET キャッシュなし 🔵

**Given**: キャッシュなし
**When**: GET /study-notes
**Then**: 200 OK + data: null

### テストケース5: POST バリデーションエラー 🔵

**Given**: source_type が不正
**When**: POST /study-notes/generate
**Then**: 400 VALIDATION_ERROR

**テストファイル**: `backend/tests/unit/test_study_notes_handler.py`
