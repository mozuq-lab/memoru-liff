## タスク概要

`StudyNotesService` にデッキ/タグからカードを取得し、カード数バリデーション（5枚以上）と代表カード選択（100枚超の場合）のロジックを実装する。

## 実装詳細

### 1. デッキからカード取得 🔵

**信頼性**: 🔵 *既存card_service・cardsテーブル設計より*

cards テーブルを Query で user_id で取得し、deck_id でフィルタ。

**実装ファイル**: `backend/src/services/study_notes_service.py`

### 2. タグからカード取得 🔵

**信頼性**: 🔵 *REQ-ASN-002・cardsテーブルのtags属性より*

cards テーブルを Query で user_id で取得し、tags に指定タグを含むカードをフィルタ。

### 3. カード数バリデーション 🔵

**信頼性**: 🔵 *REQ-ASN-101・ヒアリングQ5「5枚以上」より*

5枚未満の場合は `InsufficientCardsError` を発生させる。

### 4. 代表カード選択 🔵

**信頼性**: 🔵 *REQ-ASN-102・設計ヒアリングで選択アルゴリズムが確定*

100枚超の場合、reviews テーブルから ease_factor を取得し、ease_factor の昇順ソート（難易度の高いカード優先）で上位100枚を選択する。ease_factor が同値の場合は card_id の昇順で安定ソートする。

## 単体テスト要件

### テストケース1: デッキからカード取得（正常系） 🔵

**Given**: user_id, deck_id（10枚のカード）
**When**: get_cards_for_deck
**Then**: 10枚のCardForNotes が返却される

### テストケース2: カード数5枚未満でエラー 🔵

**Given**: deck_id（3枚のカード）
**When**: get_cards_for_deck
**Then**: InsufficientCardsError が発生

### テストケース3: カード数ちょうど5枚 🔵

**Given**: deck_id（5枚のカード）
**When**: get_cards_for_deck
**Then**: 正常に5枚が返却される

### テストケース4: 100枚超で代表選択 🔵

**Given**: deck_id（150枚のカード、各カードにease_factorが設定済み）
**When**: get_cards_for_deck
**Then**: ease_factor昇順で上位100枚が選択されて返却される

### テストケース5: タグからカード取得（複数デッキ横断） 🔵

**Given**: tag="english"（デッキA: 5枚, デッキB: 8枚）
**When**: get_cards_for_tag
**Then**: 13枚のCardForNotes が返却される

**テストファイル**: `backend/tests/unit/test_study_notes_service.py`
