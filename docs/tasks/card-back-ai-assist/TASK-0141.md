## 主要実装項目

### 「AI で補足」ボタン追加 🔵

**信頼性**: 🔵 *要件定義 REQ-001・設計文書より*

`frontend/src/components/CardForm.tsx` に追加:
- 「AI で補足」ボタンを保存ボタンの横に配置
- ボタン有効条件: `!isRefining && !isSaving && (front.trim() || back.trim())`

**実装ファイル**: `frontend/src/components/CardForm.tsx`

### 状態管理 🔵

**信頼性**: 🔵 *dataflow.md フロントエンド状態管理フローより*

状態変数追加:
- `isRefining: boolean` — AI 処理中フラグ
- `refineError: string | null` — エラーメッセージ

処理フロー:
1. ボタン押下 → isRefining=true, refineError=null
2. refineCard API 呼び出し
3. 成功 → front/back テキストエリア更新、isRefining=false
4. 失敗 → refineError にエラーメッセージ設定、元テキスト維持、isRefining=false

### エラー表示 🔵

**信頼性**: 🔵 *設計文書エラーハンドリングテーブルより*

- タイムアウト: 「AIの処理がタイムアウトしました。再度お試しください」
- レート制限: 「リクエスト制限に達しました。しばらくお待ちください」
- AI 障害: 「AIサービスが一時的に利用できません」
- バリデーション: 「入力内容を確認してください」
