## 主要実装項目

### POST /cards/refine ハンドラー 🔵

**信頼性**: 🔵 *設計文書 architecture.md・dataflow.md メインフローより*

`backend/src/api/handlers/ai_handler.py` に追加:
- `@app.post("/cards/refine")` デコレーター
- `get_user_id_from_context()` で JWT 認証
- `RefineCardRequest` でリクエストボディをバリデーション
- `ai_service.refine_card(front, back, language)` 呼び出し
- `RefineCardResponse` 構築（processing_time_ms を計測）
- `map_ai_error_to_http()` で AI エラーを HTTP ステータスにマッピング

**実装ファイル**: `backend/src/api/handlers/ai_handler.py`

### エラーハンドリング 🔵

**信頼性**: 🔵 *設計文書エラーハンドリングテーブルより*

既存の `map_ai_error_to_http()` を活用:
- AITimeoutError → 504
- AIRateLimitError → 429
- AIProviderError → 503
- AIParseError → 500
- バリデーションエラー → 400
