## 主要実装項目

### AIService Protocol 拡張 🔵

**信頼性**: 🔵 *設計文書 architecture.md バックエンドコンポーネント構成より*

`backend/src/services/ai_service.py` に追加:
- `RefineResult` dataclass: `refined_front: str`, `refined_back: str`
- `AIService` Protocol に `refine_card(front: str, back: str, language: str) -> RefineResult` メソッド追加

**実装ファイル**: `backend/src/services/ai_service.py`

### StrandsAIService refine_card 実装 🔵

**信頼性**: 🔵 *既存 generate_cards 実装パターンより*

`backend/src/services/strands_service.py` に追加:
- `refine_card()` メソッド: プロンプト構築 → Strands Agent 呼び出し → レスポンスパース
- `_parse_refine_result()` メソッド: JSON レスポンスから refined_front, refined_back を抽出
- エラーハンドリング: AITimeoutError, AIProviderError, AIParseError

**実装ファイル**: `backend/src/services/strands_service.py`

## 基本的なテスト要件

### refine_card 正常系テスト 🔵

**信頼性**: 🔵 *要件定義 REQ-002, REQ-006, REQ-007 より*

- 表面・裏面の両方を送信した場合、両方の改善結果が返ること
- 表面のみ送信した場合、refined_front のみ有効な値、refined_back が空文字であること
- 裏面のみ送信した場合、refined_back のみ有効な値、refined_front が空文字であること

### _parse_refine_result テスト 🟡

**信頼性**: 🟡 *既存 _parse_cards パターンから妥当な推測*

- 正しい JSON レスポンスのパースが成功すること
- 不正な JSON の場合に AIParseError が発生すること
- refined_front/refined_back キーが欠落している場合のエラーハンドリング
