## タスク概要

api.ts の 401 レスポンス時のトークンリフレッシュ + リトライ処理に _isRetry フラグを導入し、再帰呼び出しを最大1回に制限。無限再帰リスクを解消。

## 依存タスク

- **前提タスク**: なし
- **後続タスク**: なし

## 完了条件

- [x] request メソッドに _isRetry パラメータを追加
- [x] 401 レスポンス時のリトライを1回に制限
- [x] 2回目の 401 はエラーを投げることを確認
- [x] テストカバレッジ 80% 以上を維持

---

## 主要実装項目

### 1. request メソッドに _isRetry = false パラメータ追加 🔵

**信頼性**: 🔵 *M-01: 両レビュアー一致*

request メソッドのシグネチャに _isRetry = false パラメータを追加する。

**実装ファイル**: `frontend/src/services/api.ts`

### 2. 401 && !_isRetry 時のみ refreshToken + リトライ 🔵

**信頼性**: 🔵

401 レスポンスかつ _isRetry が false の場合のみ、トークンリフレッシュとリトライを実行する。

**実装ファイル**: `frontend/src/services/api.ts`

### 3. リトライ呼び出しで _isRetry=true を渡す 🔵

**信頼性**: 🔵

リトライ時の再帰呼び出しで _isRetry=true を指定し、2回目以降のリトライを防止する。

**実装ファイル**: `frontend/src/services/api.ts`
