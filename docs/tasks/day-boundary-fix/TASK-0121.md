## 実装詳細

### test_review_service.py: 弱いアサーション `assert A or B` を単一期待値に修正

**現状** (line ~235):
```python
assert actual_due_date == expected.date().isoformat() or actual_due_date == "2024-06-16"
```

**問題**: `assert A or B` は2つの矛盾する期待値を受け入れており、テストの検証力が弱い。

**修正方針**:
1. テストの前提条件（mock datetime、interval、day_start_hour、timezone）を確認
2. `calculate_next_review_boundary()` の結果を手計算で期待値を特定
3. 現行仕様（`due_date` は UTC 基準の `next_review_at.date().isoformat()`）に合わせて単一期待値を設定
4. `assert actual_due_date == "YYYY-MM-DD"` に修正

**注意**: 修正時にテストの前提条件（mock 値）を確認し、正確な期待値を算出すること。
