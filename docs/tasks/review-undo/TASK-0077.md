## 主要実装項目

### 1. handleUndo関数実装 🔵

**信頼性**: 🔵 *dataflow.md セクション3（取り消しフロー）より*

ReviewPage内に `handleUndo(index: number)` を実装:
1. `setIsUndoing(true)`, `setUndoingIndex(index)`
2. `reviewsApi.undoReview(cardId)` 呼び出し
3. 成功時: `reviewResults[index].type = 'undone'` に更新
4. `setRegradeCardIndex(index)`, `setIsComplete(false)`, `setIsFlipped(false)`
5. 失敗時: エラーメッセージ表示、状態復元
6. `setIsUndoing(false)`

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`

### 2. 再採点モード表示ロジック 🔵

**信頼性**: 🔵 *dataflow.md セクション4（再採点フロー）・設計ヒアリングQ2より*

`regradeCardIndex !== null` の場合:
- 対象カードの情報でFlipCard + GradeButtonsを表示
- プログレスバーは非表示（1枚のみの再採点）
- handleGrade で再採点結果を反映し、完了画面に戻る

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`

### 3. 再採点時のhandleGrade修正 🟡

**信頼性**: 🟡 *dataflow.md セクション4から妥当な推測*

再採点モード（`regradeCardIndex !== null`）の場合:
- `submitReview` API呼び出し
- 成功時: `reviewResults[regradeCardIndex]` を更新（grade, nextReviewDate, type='graded'）
- `setRegradeCardIndex(null)`, `setIsComplete(true)`
- 失敗時: エラー表示、`setRegradeCardIndex(null)`, `setIsComplete(true)`（undone状態のまま）

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`

### 4. エラーハンドリング 🟡

**信頼性**: 🟡 *dataflow.md セクション5（エラーハンドリングフロー）から妥当な推測*

- Undo APIエラー: トースト/アラートでエラー表示、完了画面に留まる
- 再採点APIエラー: エラー表示、カードはundone状態のまま完了画面に戻る

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`
