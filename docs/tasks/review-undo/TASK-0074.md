## タスク概要

`POST /reviews/{cardId}/undo` エンドポイントを実装する。カードの最新review_historyエントリからSRSパラメータの前回値を取得し、カードのease_factor/interval/repetitions/next_review_atを復元する。review_historyの最新エントリは削除し、reviewsテーブルのレコードは保持する。

`backend/src/models/review.py` に `UndoRestoredState` と `UndoReviewResponse` を追加。

**実装ファイル**: `backend/src/models/review.py`

### 2. ReviewService.undo_review()メソッド 🔵

**信頼性**: 🔵 *api-endpoints.md バックエンド実装仕様・dataflow.md セクション6より*

処理フロー:
1. `card_service.get_card(user_id, card_id)` でカード取得・所有権確認
2. `review_history` 確認（空なら400エラー）
3. 最新エントリから `ease_factor_before`, `interval_before`, `repetitions_before`, `next_review_at_before` を取得
4. DynamoDB UpdateItem でSRSパラメータを復元 + review_history最新エントリ削除
5. `UndoReviewResponse` を返却

**実装ファイル**: `backend/src/services/review_service.py`

**実装ファイル**: `backend/src/api/handler.py`

## 注意事項

- DynamoDB UpdateItem で SRSパラメータ復元と review_history 最新エントリ削除を1回のオペレーションで行う
- ease_factorの最小値1.3（SM-2制約）を下回らないよう注意
- 認証・認可は既存パターン（JWT Bearer Token + get_user_id_from_context）を使用
