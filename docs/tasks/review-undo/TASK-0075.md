## 主要実装項目

### 1. 型定義追加 🔵

**信頼性**: 🔵 *interfaces.ts SessionCardResult・UndoReviewResponseより*

`frontend/src/types/card.ts` に以下の型を追加:
- `SessionCardResultType = 'graded' | 'skipped' | 'undone'`
- `SessionCardResult { cardId, front, grade, nextReviewDate, type }`
- `UndoReviewResponse { card_id, restored, undone_at }`
- `UndoRestoredState { ease_factor, interval, repetitions, due_date }`

**実装ファイル**: `frontend/src/types/card.ts`

### 2. API関数追加 🔵

**信頼性**: 🔵 *api-endpoints.md フロントエンド実装仕様より*

`ApiClient`クラスに `undoReview(cardId)` メソッドを追加。
`reviewsApi` オブジェクトに `undoReview` を追加。

**実装ファイル**: `frontend/src/services/api.ts`

### 3. ReviewPage結果蓄積ロジック 🔵

**信頼性**: 🔵 *architecture.md ReviewPage修正セクション・dataflow.md セクション1より*

- `useState<SessionCardResult[]>` で `reviewResults` を管理
- `handleGrade` 内で `ReviewResponse` の情報を `SessionCardResult` に変換して蓄積
- `handleSkip` 内でスキップ情報を `SessionCardResult` に蓄積
- `ReviewComplete` に `results` props として渡す

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`
