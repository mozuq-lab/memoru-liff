## タスク概要

handler.py（約1024行・25関数）をドメイン別にルーター分割する。Lambda Powertools の `APIGatewayHttpResolver.include_router()` を使用し、handler.py はルーターとして維持しつつ、ドメインハンドラを `handlers/` 配下に分割する。共通関数は `shared.py` に切り出す。

## 主要実装項目

### handler.py ルーター化 🔵

**信頼性**: 🔵 *architecture.md セクション1 + ヒアリング回答より*

- `APIGatewayHttpResolver` + `include_router()` でルーター化
- `lambda_handler` 関数は `handler.py` に残す

**実装ファイル**: `backend/src/api/handler.py`

### ドメインハンドラ作成 🔵

**信頼性**: 🔵 *architecture.md セクション1 + dataflow.md フロー6 より*

- `handlers/__init__.py` 作成
- `user_handler.py`: `/users/me`, `/users/link-line`, `/users/me/settings`, `/users/me/unlink-line`
- `cards_handler.py`: `/cards` (GET/POST), `/cards/<id>` (GET/PUT/DELETE)
- `decks_handler.py`: `/decks` (GET/POST), `/decks/<id>` (PUT/DELETE)
- `review_handler.py`: `/cards/due` (GET), `/reviews/<id>` (POST), `/reviews/<id>/undo` (POST)

**実装ファイル**: `backend/src/api/handlers/*.py`
