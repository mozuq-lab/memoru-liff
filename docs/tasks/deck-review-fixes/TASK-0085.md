## タスク概要

`card_service.py` の `update_card` メソッドで `deck_id` の null と未送信を区別できない問題を修正する。Sentinel 値パターン（`_UNSET = object()`）を導入し、`deck_id=None`（明示的クリア → REMOVE）と `deck_id=_UNSET`（未送信 → 変更なし）を区別する。DynamoDB の UpdateExpression で SET と REMOVE を組み合わせる。

## 主要実装項目

### Sentinel 値定義 🔵

**信頼性**: 🔵 *architecture.md セクション2 より*

- `_UNSET = object()` を定義（services 共通モジュールまたは card_service.py 内）
- `update_card` のデフォルト引数を `deck_id=_UNSET` に変更

**実装ファイル**: `backend/src/services/card_service.py`

### UpdateExpression 構築修正 🔵

**信頼性**: 🔵 *architecture.md セクション2・DynamoDB 技術制約より*

- `if deck_id is None:` → `remove_parts.append("deck_id")`
- `elif deck_id is not _UNSET:` → `update_parts.append("deck_id = :deck_id")`
- SET + REMOVE を組み合わせた UpdateExpression 構築

**実装ファイル**: `backend/src/services/card_service.py`

### ハンドラ側 null/未送信 変換 🔵

**信頼性**: 🔵 *architecture.md セクション2 + interfaces.ts より*

- `cards_handler.py` で JSON ボディを解析
- `"deck_id": null` → `deck_id=None` として service 呼び出し
- `"deck_id"` キーなし → `deck_id=_UNSET`（デフォルト引数）

**実装ファイル**: `backend/src/api/handlers/cards_handler.py`
