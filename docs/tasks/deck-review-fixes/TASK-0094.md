## タスク概要

`DeckFormModal` の edit モードで変更されたフィールドのみを API に送信するよう修正する。フォーム初期値を保持し、保存時に各フィールドの変更を検出して差分だけを payload に含める。空文字 → null（クリア）、選択解除 → null（クリア）の変換も行う。

- **後続タスク**: [TASK-0098](TASK-0098.md)

## 完了条件

- [x] edit モードでフォーム初期値を保持
- [x] 保存時に各フィールドの変更を検出
- [x] 変更されたフィールドのみを API に送信
- [x] description 空文字 → null（クリア）変換

## 主要実装項目

### UpdateDeckRequest 型修正 🔵

**信頼性**: 🔵 *interfaces.ts・REQ-105, REQ-106 より*

- `description?: string` → `description?: string | null`
- `color?: string` → `color?: string | null`

**実装ファイル**: `frontend/src/types/deck.ts`

### 差分検出ロジック 🔵

**信頼性**: 🔵 *architecture.md セクション8・dataflow.md フロー7 より*

- 初期値を `DeckFormInitialValues` として保持
- 保存時: `name !== initial.name` → `payload.name = name`
- `description !== initial.description` → `payload.description = description || null`
- `color !== initial.color` → `payload.color = color || null`

**実装ファイル**: `frontend/src/components/DeckFormModal.tsx`

## 基本的なテスト要件

### 差分送信テスト 🔵

**信頼性**: 🔵 *REQ-202 より*

- name のみ変更 → name のみ送信
- description をクリア → `{ description: null }` 送信
- 変更なし → 空 payload（または送信しない）
