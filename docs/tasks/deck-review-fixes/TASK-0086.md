## タスク概要

`deck_service.py` の `create_deck` メソッドで、ユーザーあたりのデッキ数制限（50）をアトミックに検証する。現状の非アトミックなチェック（Query → PutItem が別操作）をQuery(Select=COUNT) + PutItem with ConditionExpression + 作成後カウント検証の3段階に変更し、レースコンディションを防止する。

## 依存タスク

- **前提タスク**: [TASK-0084: handler.py ルーター分割](TASK-0084.md)
- **後続タスク**: [TASK-0087](TASK-0087.md)

## 完了条件

- [x] `create_deck` で楽観的チェック（Query COUNT ≥ 50 → DeckLimitExceededError）
- [x] PutItem に `ConditionExpression="attribute_not_exists(user_id) AND attribute_not_exists(deck_id)"` 追加
- [x] 作成後カウント検証（> 50 → DeleteItem ロールバック + DeckLimitExceededError）
- [x] `ConditionalCheckFailedException` → `DeckLimitExceededError` 変換
- [x] テスト: 正常作成、上限到達、レース検出（mock）の3パターン
