## タスク概要

`UpdateCardRequest` の `deck_id` 型を `string | null | undefined` に変更し、`CardDetailPage` で「未分類」選択時に `{ deck_id: null }` を API に送信できるようにする。バックエンド TASK-0085 の Sentinel パターンと組み合わせて、3層（型定義 → 送信ロジック → バックエンド処理）の修正を完成させる。

- **後続タスク**: [TASK-0095](TASK-0095.md), [TASK-0097](TASK-0097.md)

## 完了条件

- [ ] `UpdateCardRequest.deck_id` の型を `string | null | undefined` に変更
- [ ] `CardDetailPage` で「未分類」選択 → `{ deck_id: null }` を送信

- [ ] テスト: null 送信、値送信、未送信の各パターン

## 主要実装項目

### UpdateCardRequest 型修正 🔵

**信頼性**: 🔵 *interfaces.ts・REQ-002 より*

- `deck_id?: string` → `deck_id?: string | null`
- undefined: 未送信（変更なし）、null: 明示的クリア、string: 値変更

**実装ファイル**: `frontend/src/types/card.ts`

### CardDetailPage null 送信 🔵

**信頼性**: 🔵 *architecture.md セクション2・dataflow.md フロー2 より*

- デッキ選択で「未分類」を選んだ場合、`deck_id: null` を送信
- 既存の保存処理を修正

**実装ファイル**: `frontend/src/pages/CardDetailPage.tsx`

## 基本的なテスト要件

### null 送信テスト 🔵

**信頼性**: 🔵 *REQ-002, REQ-103, REQ-104, EDGE-101 より*
