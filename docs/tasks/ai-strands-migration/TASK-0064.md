### 1. docker-compose.yaml に Ollama サービス追加

#### Ollama サービス定義

`backend/docker-compose.yaml` に以下の Ollama サービスを追加する:

```yaml
ollama:
  image: ollama/ollama:latest
  container_name: memoru-ollama
  ports:
    - "11434:11434"
  environment:
    - OLLAMA_HOST=0.0.0.0:11434
  volumes:
    - ollama-data:/root/.ollama
  networks:
    - memoru-network
  healthcheck:
    test: ["CMD", "curl", "-f", "http://localhost:11434/api/tags"]
    interval: 10s
    timeout: 5s
    retries: 5
    start_period: 30s
```

### 3. Makefile にコマンド追加

#### `backend/Makefile` に新規ターゲット追加

```makefile

local-ollama:
	docker-compose up -d ollama
	@echo "Waiting for Ollama service to be ready..."
```

## Notes

- Ollama の初回モデルダウンロードには時間がかかる（**数分～数十分**）。ローカル環境のネットワーク速度に依存する
- docker-compose のネットワーク名は既存の `memoru-network` を使用して、DynamoDB と Keycloak との相互通信を確保
- Ollama サービスは healthcheck を設定して起動完了を確認し、他のサービスとの起動順序トラブルを回避
- llama3.2 は十分なモデルサイズ（9B）で、ローカル開発用のメモリ効率とパフォーマンスのバランスが良い
- `OLLAMA_HOST` を `0.0.0.0:11434` に設定することで、コンテナ外からのアクセスを許可（ローカル開発用）
