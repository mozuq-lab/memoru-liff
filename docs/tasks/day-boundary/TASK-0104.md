## タスク概要

レビュー送信時にユーザー設定を取得し、`calculate_next_review_boundary()` で `next_review_at` を正規化する。

## 依存タスク

- **前提タスク**: [TASK-0102: SRS 正規化関数の実装](TASK-0102.md), [TASK-0103: ユーザー設定モデル拡張](TASK-0103.md)
- **後続タスク**: なし

## 完了条件

- [ ] `review_handler` が UserService でユーザー設定を取得する
- [ ] `review_service.submit_review()` が正規化された next_review_at を保存する
- [ ] 既存のクエリロジックが変更されていない
- [ ] 単体テストが全て通る

## 主要実装項目

### 1. `review_handler.py` に `UserService` をインポート・インスタンス化 🔵

**信頼性**: 🔵 *architecture.md コンポーネント設計3より*

`cards_handler.py` と同じパターンで `UserService` をインポート・インスタンス化する。

**実装ファイル**: `backend/src/api/handlers/review_handler.py`

### 2. `submit_review()` ハンドラーでユーザー設定を取得し渡す 🔵

**信頼性**: 🔵 *architecture.md コンポーネント設計3より*

`submit_review()` ハンドラーでユーザー設定を取得し、`review_service.submit_review()` に渡す。

**実装ファイル**: `backend/src/api/handlers/review_handler.py`

### 3. `review_service.submit_review()` に正規化ロジック追加 🔵

**信頼性**: 🔵 *architecture.md コンポーネント設計3より*

`review_service.submit_review()` に `user_timezone`, `day_start_hour` パラメータを追加。`calculate_sm2()` 後に `calculate_next_review_boundary()` で正規化し、正規化済み `next_review_at` でカード更新する。

**実装ファイル**: `backend/src/services/review_service.py`
