## 主要実装項目

### 1. CardForm への ReferenceEditor 統合 🔵

**信頼性**: 🔵 *既存 CardForm のパターンに準拠*

**実装ファイル**: `frontend/src/components/CardForm.tsx`

CardForm のフォーム state に references を追加し、ReferenceEditor を配置する:

```tsx
import { ReferenceEditor } from "./ReferenceEditor";

// フォーム state に references を追加
const [references, setReferences] = useState<Reference[]>(
  initialCard?.references ?? []
);

// フォーム送信時に references を含める
const handleSubmit = () => {
  onSubmit({
    // ... 既存フィールド ...
    references,
  });
};

// JSX 内に ReferenceEditor を配置
<ReferenceEditor references={references} onChange={setReferences} />
```

### 2. CardDetailPage への ReferenceDisplay 統合 🔵

**信頼性**: 🔵 *既存 CardDetailPage のパターンに準拠*

**実装ファイル**: `frontend/src/pages/CardDetailPage.tsx`

カード詳細の表示セクションに ReferenceDisplay を追加:

```tsx
import { ReferenceDisplay } from "../components/ReferenceDisplay";

// カード情報表示セクション内
<ReferenceDisplay references={card.references} />
```

編集モードでは ReferenceEditor を表示する（CardForm 経由で対応済みの場合はそちらを利用）。

### 3. ReviewPage への ReferenceDisplay 統合 🔵

**信頼性**: 🔵 *既存 ReviewPage のパターンに準拠*

**実装ファイル**: `frontend/src/pages/ReviewPage.tsx`

カード裏面（回答表示後）のセクションに ReferenceDisplay を追加:

```tsx
import { ReferenceDisplay } from "../components/ReferenceDisplay";

// カード裏面セクション内
{showAnswer && (
  <>
    {/* ... 既存の回答表示 ... */}
    <ReferenceDisplay references={card.references} />
  </>
)}
```

### 4. 後方互換性対応 🔵

**信頼性**: 🔵 *TypeScript のデフォルト値で対応*

API レスポンスで references が undefined の場合に備え、各所で `card.references ?? []` のフォールバックを適用する。ReferenceDisplay は空配列で何も表示しないため、既存カードでも問題なく動作する。
