## タスク概要

バックエンド・フロントエンドの環境変数ファイル（`.env`, `.env.example` 等）を新しい OIDC 汎用名に更新する。Keycloak / Cognito それぞれの設定例をコメントで併記し、切り替え手順を明確にする。

## 依存タスク

- **前提タスク**: [TASK-0123](TASK-0123.md), [TASK-0124](TASK-0124.md)
- **後続タスク**: なし

## 完了条件

- [ ] フロントエンドの `.env` / `.env.example` が新しい環境変数名（`VITE_OIDC_*`）に更新されている
- [ ] バックエンドの SAM パラメータオーバーライド設定（`samconfig.toml` 等）が新しいパラメータ名に更新されている（存在する場合）
- [ ] Keycloak / Cognito 両方の設定例がコメントで併記されている
- [ ] `make local-api` でバックエンドが起動できる（設定ファイルが正しいことの確認）
- [ ] `npm run dev` でフロントエンドが起動できる

## 主要実装項目

### 1. フロントエンド環境変数ファイルの更新 🟡

**信頼性**: 🟡 *設計文書から妥当な推測（`.env` ファイルの正確な構成は未確認）*

**実装ファイル**: `frontend/.env`, `frontend/.env.example` 等

```env
# === OIDC Provider Configuration ===
# Keycloak の場合:
VITE_OIDC_AUTHORITY=https://keycloak.example.com/realms/memoru
VITE_OIDC_CLIENT_ID=liff-client

# Cognito の場合:
# VITE_OIDC_AUTHORITY=https://cognito-idp.ap-northeast-1.amazonaws.com/ap-northeast-1_XXXXXXXXX
# VITE_OIDC_CLIENT_ID=xxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 2. バックエンド SAM 設定の更新 🟡

**信頼性**: 🟡 *設計文書から妥当な推測（`samconfig.toml` の構成は未確認）*

**実装ファイル**: `backend/samconfig.toml`（存在する場合）

- `KeycloakIssuer` → `OidcIssuer` のパラメータオーバーライド更新
- `OidcAudience` のパラメータオーバーライド追加

### 3. Makefile の確認・更新 🟡

**信頼性**: 🟡 *妥当な推測（Makefile 内で環境変数を参照している可能性）*

**実装ファイル**: `backend/Makefile`

- `local-api` ターゲット等で `KEYCLOAK_ISSUER` を参照している箇所があれば `OIDC_ISSUER` に更新
