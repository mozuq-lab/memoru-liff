# TASK-0124: frontend OIDC 設定汎用化

**タスクタイプ**: TDD
**推定工数**: 3時間
**フェーズ**: Phase 1 - OIDC汎用化
**信頼性レベル**: 🔵 *設計文書 architecture.md のファイル別変更設計より*

## 関連文書

- **概要**: [📋 overview.md](overview.md)

## タスク概要

`frontend/src/config/oidc.ts` の Keycloak 固有の環境変数名・URL構築ロジックを汎用的な OIDC 設定に変更する。環境変数 `VITE_KEYCLOAK_*` → `VITE_OIDC_*` へのリネームと、`authority` URL の直接指定化を行う。フロントエンドテストも同時に更新する。

## 依存タスク

- **前提タスク**: [TASK-0123](TASK-0123.md)（バックエンド側の audience パラメータ化と整合を取るため）
- **後続タスク**: [TASK-0125](TASK-0125.md)

## 完了条件

- [x] `oidc.ts` の環境変数が `VITE_OIDC_AUTHORITY`, `VITE_OIDC_CLIENT_ID` に変更されている
- [x] `authority` が `VITE_OIDC_AUTHORITY` の値をそのまま使用している（Keycloak 固有の URL 構築を除去）
- [x] `validateOidcConfig()` が新しい環境変数名でバリデーションしている
- [x] フロントエンドテストが新しい環境変数名に対応している
- [x] `npm run test` が全件パスする
- [x] `npm run type-check` がエラーなし

## 主要実装項目

### 1. 環境変数のリネーム 🔵

**信頼性**: 🔵 *設計文書 architecture.md「環境変数変更」セクションより*

**実装ファイル**: `frontend/src/config/oidc.ts`

```typescript
// Before: 3変数（Keycloak固有）
const KEYCLOAK_URL = import.meta.env.VITE_KEYCLOAK_URL || '';
const KEYCLOAK_REALM = import.meta.env.VITE_KEYCLOAK_REALM || '';
const KEYCLOAK_CLIENT_ID = import.meta.env.VITE_KEYCLOAK_CLIENT_ID || '';

// After: 2変数（汎用OIDC）
const OIDC_AUTHORITY = import.meta.env.VITE_OIDC_AUTHORITY || '';
const OIDC_CLIENT_ID = import.meta.env.VITE_OIDC_CLIENT_ID || '';
```

### 2. authority URL 構築の簡素化 🔵

**信頼性**: 🔵 *設計文書 architecture.md「OIDC 設定変更」セクションより*

**実装ファイル**: `frontend/src/config/oidc.ts`

```typescript
// Before: Keycloak固有のURL構築
authority: `${KEYCLOAK_URL}/realms/${KEYCLOAK_REALM}`,

// After: 環境変数の値をそのまま使用
authority: OIDC_AUTHORITY,
```

### 3. バリデーション更新 🔵

**信頼性**: 🔵 *設計文書 architecture.md「バリデーション変更」セクションより*

**実装ファイル**: `frontend/src/config/oidc.ts`

- `validateOidcConfig()` の `requiredVars` を新しい環境変数名に更新
- 変数が3→2に減るため、バリデーション対象も変更

### 4. フロントエンドテストの更新 🟡

**信頼性**: 🟡 *設計文書「テストファイルの影響」セクションから妥当な推測*

**実装ファイル**: `frontend/src/config/__tests__/` 配下

- テストで旧環境変数名（`VITE_KEYCLOAK_*`）を参照している箇所を更新
- `validateOidcConfig()` のテストケースを新しい環境変数名に更新
- authority URL の期待値を更新

### 5. コメント・JSDoc の更新 🟡

**信頼性**: 🟡 *既存コードのコメントから妥当な推測*

**実装ファイル**: `frontend/src/config/oidc.ts`

- ファイル先頭のコメント（「Keycloak OIDC設定」→「OIDC設定」）
- 各プロパティのコメントから Keycloak 固有の記述を除去
