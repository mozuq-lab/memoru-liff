## タスク概要

`backend/template.yaml` の Keycloak 固有のパラメータ名・環境変数名を汎用的な OIDC 名称にリネームし、`audience` をパラメータ化する。併せてバックエンドテストで旧環境変数名を参照している箇所を更新する。

## 依存タスク

- **前提タスク**: なし
- **後続タスク**: [TASK-0124](TASK-0124.md), [TASK-0125](TASK-0125.md)

## 完了条件

- [x] `template.yaml` の `KeycloakIssuer` パラメータが `OidcIssuer` にリネームされている
- [x] `template.yaml` に `OidcAudience` パラメータが追加されている
- [x] JWT Authorizer の `audience` が `!Ref OidcAudience` を参照している
- [x] Lambda 環境変数 `KEYCLOAK_ISSUER` が `OIDC_ISSUER` にリネームされている
- [x] バックエンドテスト（`conftest.py` 等）で旧環境変数名を参照している箇所が更新されている
- [x] `make test` が全件パスする

---

## 主要実装項目

### 1. SAM パラメータのリネーム 🔵

**信頼性**: 🔵 *設計文書 architecture.md「パラメータ変更」セクションより*

**実装ファイル**: `backend/template.yaml`

- `KeycloakIssuer` → `OidcIssuer`（Type, Default, Description を更新）
- `OidcAudience` パラメータを新規追加（Default: `liff-client`）

### 2. JWT Authorizer の audience パラメータ化 🔵

**信頼性**: 🔵 *設計文書 architecture.md「JWT Authorizer 変更」セクションより*

**実装ファイル**: `backend/template.yaml`

- `issuer: !Ref KeycloakIssuer` → `issuer: !Ref OidcIssuer`
- `audience: - liff-client` → `audience: - !Ref OidcAudience`

### 3. Lambda 環境変数のリネーム 🔵

**信頼性**: 🔵 *設計文書 architecture.md「Lambda 環境変数変更」セクションより*

**実装ファイル**: `backend/template.yaml`

- 全 Lambda 関数の `KEYCLOAK_ISSUER` → `OIDC_ISSUER`

### 4. バックエンドテストの更新 🟡

**信頼性**: 🟡 *設計文書「テストファイルの影響」セクションから妥当な推測*

**実装ファイル**: `backend/tests/conftest.py` 他

- テストで `KEYCLOAK_ISSUER` を参照している箇所を `OIDC_ISSUER` に更新
- モック issuer URL のテストデータを確認・更新
