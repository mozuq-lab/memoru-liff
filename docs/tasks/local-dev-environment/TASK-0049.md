## 主要実装項目

### 1. 既存テストのカバレッジ確認 🔵

**信頼性**: 🔵 _計画ファイルより_

現在の `get_user_id_from_context()` テストを確認し、dev フォールバックパスがカバーされているか検証する。

**確認ファイル**: `backend/tests/test_handler.py`

### 2. dev フォールバックパス テスト追加 🔵

**信頼性**: 🔵 _設計文書 architecture.md より_

以下のテストケースが不足している場合に追加する:

1. **正常系**: `ENVIRONMENT=dev` + 有効な JWT の Authorization ヘッダー → sub が返る
2. **異常系**: `ENVIRONMENT=dev` + Authorization ヘッダーなし → 401
3. **異常系**: `ENVIRONMENT=dev` + `Bearer ` プレフィックスなし → 401
4. **異常系**: `ENVIRONMENT=dev` + base64 デコード不可な JWT → 401
5. **異常系**: `ENVIRONMENT=dev` + sub クレームなしの JWT → 401
6. **セキュリティ**: `ENVIRONMENT=prod` + JWT ヘッダーあり → フォールバック無効 → 401
7. **優先順位**: authorizer context あり + JWT ヘッダーあり → authorizer context を優先

**テストファイル**: `backend/tests/test_handler.py`

### get_user_id_from_context() dev フォールバック テスト 🔵

**信頼性**: 🔵 _設計文書・要件定義より_
