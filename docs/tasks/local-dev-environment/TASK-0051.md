## タスク概要

JWT フォールバック（TASK-0049）と DynamoDB local SigV4 修正（TASK-0050）の両方が完了した状態で、ローカル開発環境のエンドツーエンド動作を確認する。フロントエンドからバックエンド API を経由して DynamoDB local にデータを読み書きできることを検証する。

- **後続タスク**: なし

## 主要実装項目

### 1. ローカル環境フル起動テスト 🔵

**信頼性**: 🔵 *起動手順は Makefile・dataflow.md に記載*

```bash
# ターミナル1: インフラ起動
cd backend && make local-all

# ターミナル2: API 起動
cd backend && make local-api

# ターミナル3: フロントエンド起動
cd frontend && npm run dev
```

### 2. 認証フロー E2E 確認 🔵

**信頼性**: 🔵 *dataflow.md 認証フローより*

4. ホーム画面が表示される

## 注意事項

- Keycloak の初回起動は約60秒かかる（healthcheck の start_period 設定）
- DynamoDB local のテーブルは setup-tables コンテナで自動作成される
- `make local-all-stop` で全サービスを停止できる
- Keycloak dev モードは再起動でデータリセットされる（テストユーザーは realm-local.json で再インポート）
