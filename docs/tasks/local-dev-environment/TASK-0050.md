## タスク概要

`amazon/dynamodb-local:latest` が boto3 の SigV4 署名付きリクエストでハングする問題を解決する。Docker イメージのバージョンを安定版に固定し、DynamoDB local が正常に動作することを確認する。

**現在の問題**: `backend/docker-compose.yaml` L2 で `amazon/dynamodb-local:latest` を使用しているが、このイメージが SigV4 署名付きリクエストに対してハングする。

## 主要実装項目

### 1. Docker Hub で安定バージョンの特定 🟡

**信頼性**: 🟡 *設計文書の候補バージョンから確認が必要*

Docker Hub の `amazon/dynamodb-local` のタグ一覧を確認し、SigV4 問題が発生しない安定バージョンを特定する。

**候補バージョン（設計文書より）**:
- `2.5.3`（2024年後半リリース）
- `2.4.0`（広く使われている安定版）

**確認方法**:
```bash
# Docker Hub でタグ一覧を確認
docker pull amazon/dynamodb-local:{version}
docker run --rm -p 8000:8000 amazon/dynamodb-local:{version} -jar DynamoDBLocal.jar -sharedDb

# 別ターミナルで接続テスト
aws dynamodb list-tables --endpoint-url http://localhost:8000
python -c "import boto3; db = boto3.resource('dynamodb', endpoint_url='http://localhost:8000'); print(list(db.tables.all()))"
```

### 2. docker-compose.yaml のイメージ変更 🔵

**信頼性**: 🔵 *設計文書・ユーザヒアリングより方針確定済み*

**変更ファイル**: `backend/docker-compose.yaml`

```yaml
# 変更前
image: amazon/dynamodb-local:latest

# 変更後
image: amazon/dynamodb-local:{stable-version}
```

### 3. 動作確認 🔵

**信頼性**: 🔵 *設計文書の手順より*

1. 既存コンテナの停止・削除: `docker compose down -v`
2. 新しいイメージでの起動: `make local-db`
3. テーブル作成の確認: `aws dynamodb list-tables --endpoint-url http://localhost:8000`
4. boto3 接続テスト: Python スクリプトでテーブル一覧取得
5. CRUD 操作テスト: PutItem / GetItem / DeleteItem

### 4. フォールバック対応（必要な場合のみ） 🟡

**信頼性**: 🟡 *設計文書のフォールバック案より*

バージョン固定で解決しない場合の代替案:

1. **`-disableTelemetry` オプション**: docker-compose.yaml の command に追加
2. **ダミー認証情報**: `AWS_ACCESS_KEY_ID=local`, `AWS_SECRET_ACCESS_KEY=local` を明示設定
3. **localstack**: `localstack/localstack` の DynamoDB を使用

## 注意事項

- 既存の DynamoDB local コンテナのデータは Docker Volume (`memoru-dynamodb-data`) に保存されている。バージョン変更時は `docker compose down -v` でボリュームも削除する必要がある場合がある
- setup-tables コンテナ（aws-cli）は DynamoDB local の healthcheck 後に実行されるため、イメージ変更による影響は限定的
- dynamodb-admin コンテナは `DYNAMO_ENDPOINT: http://dynamodb-local:8000` で接続しているため、影響なし
- 既存の docker-compose.yaml の DynamoDB 以外のサービス（keycloak, dynamodb-admin, setup-tables）は変更しない（REQ-LD-404）
