## 実装詳細

### 1. ルート追加 🔵

**信頼性**: 🔵 *要件定義REQ-RV-004・architecture.md のルーティング設計より*

App.tsx（またはルーター設定ファイル）に `/review` ルートを追加する。

```typescript
<Route path="/review" element={<ReviewPage />} />
```

**実装ファイル**: `frontend/src/App.tsx`（またはルーター設定ファイル）

### 2. HomePage 変更 🔵

**信頼性**: 🔵 *要件定義REQ-RV-030・architecture.md の既存変更設計より*

「復習を始める」ボタンのリンク先を `/cards?tab=due` から `/review` に変更する。

```diff
- <Link to="/cards?tab=due">復習を始める</Link>
+ <Link to="/review">復習を始める</Link>
```

**実装ファイル**: `frontend/src/pages/HomePage.tsx`

### 3. CardsPage 変更 🔵

**信頼性**: 🔵 *要件定義REQ-RV-031・architecture.md の既存変更設計より*

復習対象タブにカードがある場合、「復習開始」ボタンを追加する。

```typescript
{activeTab === 'due' && displayCards.length > 0 && (
  <Link to="/review" className="...">復習開始</Link>
)}
```

**実装ファイル**: `frontend/src/pages/CardsPage.tsx`

### 4. ナビゲーションバー条件付き表示 🔵

**信頼性**: 🔵 *要件定義REQ-RV-401・ユーザヒアリングより*

ReviewPage（`/review` パス）ではナビゲーションバーを非表示にする。

**実装ファイル**: `frontend/src/App.tsx`（またはレイアウトコンポーネント）
