## 単体テスト要件

### テストケース1: 復習セッション全体フロー 🔵

**信頼性**: 🔵 *dataflow.md の復習セッション全体フローより*

**Given**: API が3枚の due_cards を返す
**When**: カード1をフリップ→採点(4)→カード2をフリップ→スキップ→カード3をフリップ→採点(5)
**Then**: 完了画面が表示され、「2枚復習しました」と表示される

**テストファイル**: `frontend/src/pages/__tests__/ReviewPage.test.tsx`

### テストケース2: カード1枚のみ 🟡

**信頼性**: 🟡 *EDGE-RV-101から妥当な推測*

**Given**: API が1枚の due_cards を返す
**When**: カードをフリップ→採点する
**Then**: 即座に完了画面に遷移する

### テストケース3: 全カードスキップ 🟡

**信頼性**: 🟡 *EDGE-RV-103から妥当な推測*

**Given**: API が3枚の due_cards を返す
**When**: すべてのカードをスキップする
**Then**: 完了画面が表示され、「0枚復習しました」と表示される

### テストケース4: 初期読み込みエラーからのリトライ 🟡

**信頼性**: 🟡 *REQ-RV-103・既存実装パターンから妥当な推測*

**Given**: GET /cards/due が1回目に失敗する
**When**: リトライボタンをクリックする
**Then**: 再度 API が呼ばれ、成功時にカードが表示される

### テストケース5: 戻るボタンでセッション中断 🔵

**信頼性**: 🔵 *要件定義REQ-RV-402より*

**Given**: 復習セッション中
**When**: 「戻る」ボタンをクリックする
**Then**: 前の画面に戻る

### テストケース6: ナビゲーションバー非表示 🔵

**信頼性**: 🔵 *要件定義REQ-RV-401より*

**Given**: /review ページにいる
**When**: 画面を確認する
**Then**: ナビゲーションバーが表示されていない
