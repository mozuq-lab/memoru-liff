## タスク概要

SM-2アルゴリズムの6段階（0-5）採点ボタンとスキップボタンを提供するコンポーネントを実装する。各ボタンは数字と色で区別し、送信中の無効化機能を含む。

## 実装詳細

### 1. 採点ボタン設定定数 🔵

**信頼性**: 🔵 *architecture.md のボタンデザイン表・interfaces.ts より*

GRADE_BUTTON_CONFIGS 定数を利用する（interfaces.ts に定義済み）。ただし実装時は `frontend/src/components/GradeButtons.tsx` 内にローカル定義してもよい。

| Grade | ラベル | 色 | SM-2の意味 |
|-------|--------|-----|-----------|
| 0 | 0 | red-600 | 全く覚えていない |
| 1 | 1 | orange-600 | 間違えた |
| 2 | 2 | amber-500 | 間違えたが見覚えあり |
| 3 | 3 | yellow-500 | 難しかったが正解 |
| 4 | 4 | lime-500 | やや迷ったが正解 |
| 5 | 5 | green-600 | 完璧 |

**実装ファイル**: `frontend/src/components/GradeButtons.tsx`

### 2. GradeButtons コンポーネント 🔵

**信頼性**: 🔵 *architecture.md のコンポーネント設計・interfaces.ts より*

```typescript
interface GradeButtonsProps {
  onGrade: (grade: number) => void;
  onSkip: () => void;
  disabled: boolean;
}
```

- 6つの採点ボタンを横並び（2行3列）で表示
- スキップボタンは別行に配置
- disabled 時はすべてのボタンに `opacity-50 cursor-not-allowed` を適用
- 各ボタンの説明テキストはボタン下に小さく表示（ユーザヒアリングより数字+色のみ推奨）
- タッチターゲット最小 44px（REQ-RV-404）

**実装ファイル**: `frontend/src/components/GradeButtons.tsx`

### 3. ボタンスタイリング 🟡

**信頼性**: 🟡 *architecture.md のボタンデザイン表から妥当な推測*

Tailwind CSS でボタンスタイリング。各ボタンに bg-{color} クラスを適用。ホバー・アクティブ状態のスタイルも定義。
