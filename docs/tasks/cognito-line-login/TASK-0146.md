## 主要実装項目

### 1. CognitoStackProps の拡張 🔵

**信頼性**: 🔵 *architecture.md「CognitoStackProps の拡張」セクション・REQ-006 より*

`CognitoStackProps` にオプショナルな LINE Login Props を追加する。

**実装ファイル**: `infrastructure/cdk/lib/cognito-stack.ts`

```typescript
export interface CognitoStackProps extends cdk.StackProps {
  environment: Environment;
  cognitoDomainPrefix: string;
  callbackUrls: string[];
  logoutUrls: string[];
  lineLoginChannelId?: string;
  lineLoginChannelSecret?: string;
}
```

### 2. UserPoolIdentityProviderOidc の条件付き作成 🔵

**信頼性**: 🔵 *architecture.md「UserPoolIdentityProviderOidc の追加」セクション・REQ-001, REQ-002 より*

LINE Login の Channel ID/Secret が両方指定された場合のみ IdP を作成する。

**実装ファイル**: `infrastructure/cdk/lib/cognito-stack.ts`

```typescript
let lineProvider: cognito.UserPoolIdentityProviderOidc | undefined;

if (props.lineLoginChannelId && props.lineLoginChannelSecret) {
  lineProvider = new cognito.UserPoolIdentityProviderOidc(this, 'LineLoginProvider', {
    userPool: this.userPool,
    name: 'LINE',
    clientId: props.lineLoginChannelId,
    clientSecret: props.lineLoginChannelSecret,
    issuerUrl: 'https://access.line.me',
    scopes: ['openid', 'profile'],
    endpoints: {
      authorization: 'https://access.line.me/oauth2/v2.1/authorize',
      token: 'https://api.line.me/oauth2/v2.1/token',
      userInfo: 'https://api.line.me/v2/profile',
      jwksUri: 'https://api.line.me/oauth2/v2.1/certs',
    },
    attributeMapping: {
      custom: {
        'sub': cognito.ProviderAttribute.other('sub'),
        'name': cognito.ProviderAttribute.other('name'),
        'picture': cognito.ProviderAttribute.other('picture'),
      },
    },
  });
}
```

### 3. LINE OIDC エンドポイント 🟡

**信頼性**: 🟡 *LINE Developers 公式ドキュメントから妥当な推測。デプロイ時に実動作確認が必要*

| エンドポイント | URL |
|---|---|
| Authorization | `https://access.line.me/oauth2/v2.1/authorize` |
| Token | `https://api.line.me/oauth2/v2.1/token` |
| UserInfo | `https://api.line.me/v2/profile` |
| JWKS URI | `https://api.line.me/oauth2/v2.1/certs` |
| Issuer URL | `https://access.line.me` |

### 4. UserPoolClient の supportedIdentityProviders 更新 🔵

**信頼性**: 🔵 *architecture.md「UserPoolClient の更新」セクション・REQ-004, REQ-005 より*

LINE IdP が登録されている場合、`supportedIdentityProviders` に LINE を追加する。

**実装ファイル**: `infrastructure/cdk/lib/cognito-stack.ts`

```typescript
supportedIdentityProviders: [
  cognito.UserPoolClientIdentityProvider.COGNITO,
  ...(lineProvider
    ? [cognito.UserPoolClientIdentityProvider.custom('LINE')]
    : []),
],
```

**注意**: `lineProvider` を `UserPoolClient` より前に定義する必要がある。既存の `addClient` 呼び出しを `lineProvider` 作成後に移動するか、構成を調整すること。

### 5. テスト追加 🔵

**信頼性**: 🔵 *既存テストパターン（cognito-stack.test.ts）より*

## 注意事項

- `lineProvider` の変数宣言位置に注意。`UserPoolClient` の `supportedIdentityProviders` で参照するため、IdP 作成を先に行う必要がある
- 既存の `this.userPoolClient = this.userPool.addClient(...)` の位置を `lineProvider` 定義後に移動する必要がある可能性がある
- LINE Login の `issuerUrl` が Cognito の自動検出に対応していない場合、`endpoints` の手動指定が必須（設計書で手動指定を前提としている）
- 既存のスナップショットテストは LINE Props なしの状態で更新が必要
