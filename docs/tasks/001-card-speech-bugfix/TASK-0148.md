## 主要実装項目

### 1. useEffect で userId 変化時の再読み込み 🔵

**信頼性**: 🔵 *architecture.md REQ-002 設計判断より*

**実装ファイル**: `frontend/src/hooks/useSpeechSettings.ts`

`useState` の後に以下の `useEffect` を追加:

```tsx
import { useState, useEffect } from "react";

// ... 既存コード ...

const [settings, setSettings] = useState<SpeechSettings>(() => {
  if (!userId) return { ...DEFAULT_SETTINGS };
  return loadSettings(userId);
});

// ← 追加
useEffect(() => {
  if (userId) {
    setSettings(loadSettings(userId));
  }
}, [userId]);
```

### 2. localStorage.setItem の try/catch 🟡

**信頼性**: 🟡 *loadSettings 側の try/catch パターンからの妥当な推測*

**実装ファイル**: `frontend/src/hooks/useSpeechSettings.ts`

```tsx
const updateSettings = (patch: Partial<SpeechSettings>) => {
  setSettings((prev) => {
    const next = { ...prev, ...patch };
    if (userId) {
      try {
        localStorage.setItem(`speech-settings:${userId}`, JSON.stringify(next));
      } catch {
        // Safari Private Mode / 容量超過 — state のみ更新
      }
    }
    return next;
  });
};
```
