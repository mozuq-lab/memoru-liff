## 主要実装項目

### 1. エンドツーエンドフローテスト 🔵

**信頼性**: 🔵 *受け入れ基準 TC-001~004、ユーザーストーリー1.1より*

- 通常復習 → quality 0-2 → SM-2 API → 再確認キュー追加 → 次のカード → 再確認カード表示 → 「覚えた」 → セッション完了
- 通常復習 → quality 0-2 → 再確認 → 「覚えていない」 → キュー末尾追加 → 再度表示 → 「覚えた」

### 2. エッジケーステスト 🔵

**信頼性**: 🔵 *受け入れ基準 EDGE-001, EDGE-101, EDGE-102より*

- 全カードがquality 0-2の場合：全て再確認ループに入り、全て「覚えた」で完了
- 同一カードが複数回「覚えていない」を繰り返す場合：正常にループする
- 通常カードと再確認カードの混在フロー

### 3. Undo連携統合テスト 🔵

**信頼性**: 🔵 *受け入れ基準 TC-404-01~03より*

- quality 0-2評価 → 再確認ループ → Undo → SM-2復元 → 再確認キュー除去 → regrade quality 4 → 通常完了
- quality 0-2評価 → Undo → regrade quality 2 → 再び再確認ループに入る

### 4. 回帰テスト確認 🟡

**信頼性**: 🟡 *既存テスト維持の妥当な推測*

- 既存の全フロントエンドテストが通ることを確認
- TypeScript型チェック（tsc --noEmit）が通ることを確認
- quality 3-5の通常フローが変更なく動作することを確認

## 基本的なテスト要件

### 統合フローテスト 🔵

**信頼性**: 🔵 *受け入れ基準より*

- ReviewPage全体のレンダリング → 評価 → 再確認 → 完了の一連フロー

- 完了画面での再確認結果表示の統合確認

**信頼性**: 🟡 *既存テストカバレッジから妥当な推測*

- `npm run type-check` 通過確認
- `npm test` 全テスト通過確認
- quality 3-5フロー、スキップフロー、空カードフローの回帰確認
