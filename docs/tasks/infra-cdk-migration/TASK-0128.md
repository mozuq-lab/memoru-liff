## タスク概要

既存の `infrastructure/liff-hosting/template.yaml` (358行) と同等の LIFF Hosting スタックを CDK で実装する。S3 + CloudFront + OAC による静的サイトホスティングを L2 Construct で定義し、SPA ルーティング、セキュリティヘッダー、キャッシュポリシーを設定する。

## 主要実装項目

### 1. LiffHostingStackProps インターフェース定義 🔵

**信頼性**: 🔵 *設計文書 architecture.md 環境管理パターン・パラメータマッピングより*

```typescript
interface LiffHostingStackProps extends cdk.StackProps {
  environment: 'dev' | 'staging' | 'prod';
  domainName?: string;
  certificateArn?: string;
  hostedZoneId?: string;
  apiEndpoint?: string;  // CSP 用
}
```

**実装ファイル**: `infrastructure/cdk/lib/liff-hosting-stack.ts`

### 2. S3 バケット定義 🔵

**信頼性**: 🔵 *既存テンプレート・設計文書より*

- バケット名: `memoru-liff-${env}-${accountId}`
- `removalPolicy: RETAIN`
- `versioned: true`
- L2 Construct のデフォルトで暗号化・パブリックアクセスブロック済み
- prod のみログ用バケットを別途作成

**実装ファイル**: `infrastructure/cdk/lib/liff-hosting-stack.ts`

### 3. CloudFront Distribution 定義 🔵

**信頼性**: 🔵 *既存テンプレート・設計文書より*

- `S3BucketOrigin.withOriginAccessControl(bucket)` で OAC 自動設定
- `viewerProtocolPolicy: REDIRECT_TO_HTTPS`
- SPA エラーレスポンス: 403/404 → 200 `/index.html`
- カスタムドメイン対応（オプション）

**実装ファイル**: `infrastructure/cdk/lib/liff-hosting-stack.ts`

### 4. キャッシュポリシー・セキュリティヘッダー 🔵

**信頼性**: 🔵 *既存テンプレートの CachePolicy・ResponseHeadersPolicy より*

- デフォルト: `CachingOptimized`
- `/assets/*`: 長期キャッシュ (max-age: 31536000)
- `/index.html`: キャッシュ無効 (max-age: 0)
- ResponseHeadersPolicy: CSP, HSTS, X-Frame-Options: DENY, X-Content-Type-Options: nosniff

**実装ファイル**: `infrastructure/cdk/lib/liff-hosting-stack.ts`

### 5. CfnOutput 定義 🔵

**信頼性**: 🔵 *設計文書 architecture.md Outputs セクションより*

| Output | Export 名 | 値 |
|---|---|---|
| BucketName | `memoru-{env}-liff-bucket` | S3 バケット名 |
| DistributionId | `memoru-{env}-liff-distribution-id` | CloudFront Distribution ID |
| LiffUrl | `memoru-{env}-liff-url` | LIFF アプリ URL |
| DeployCommand | — | S3 sync + CloudFront invalidation コマンド |

**実装ファイル**: `infrastructure/cdk/lib/liff-hosting-stack.ts`
