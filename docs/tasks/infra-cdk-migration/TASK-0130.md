## タスク概要

`bin/app.ts` に CDK App のエントリポイントを実装し、各スタック (Keycloak, LIFF Hosting, Cognito) を dev/staging/prod 環境ごとにインスタンス化する。Stack Props パターンで環境差異を制御する。

- [ ] `infrastructure/cdk/bin/app.ts` に全スタックのインスタンス化が実装されている
- [ ] dev 環境の全スタック (MemoruCognitoDev, MemoruKeycloakDev, MemoruLiffHostingDev) が定義されている

## 主要実装項目

### 1. CDK App 初期化とスタックインスタンス化 🔵

**信頼性**: 🔵 *設計文書 architecture.md bin/app.ts セクションより*

```typescript
const app = new cdk.App();

// dev 環境
new CognitoStack(app, 'MemoruCognitoDev', { environment: 'dev', ... });
new KeycloakStack(app, 'MemoruKeycloakDev', { environment: 'dev', ... });
new LiffHostingStack(app, 'MemoruLiffHostingDev', { environment: 'dev', ... });

// prod 環境
new CognitoStack(app, 'MemoruCognitoProd', { environment: 'prod', ... });
new KeycloakStack(app, 'MemoruKeycloakProd', { environment: 'prod', ... });
new LiffHostingStack(app, 'MemoruLiffHostingProd', { environment: 'prod', ... });
```

**実装ファイル**: `infrastructure/cdk/bin/app.ts`

### 2. 環境別パラメータ設定 🟡

**信頼性**: 🟡 *設計文書のパラメータマッピングから妥当な推測（具体的な値はデプロイ時に決定）*

各環境の具体的なパラメータ値（ドメイン名、VPC CIDR 等）をプレースホルダとして設定する。実際の値はデプロイ前にユーザーが更新する。

**実装ファイル**: `infrastructure/cdk/bin/app.ts`
