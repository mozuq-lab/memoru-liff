## 主要実装項目

### 1. Pydantic レスポンスモデル定義 🔵

**信頼性**: 🔵 *設計文書 architecture.md のモデル定義より*

以下の Pydantic v2 モデルを定義する:

- `StatsResponse` - 学習統計サマリー（総カード数、学習済み、習得済み、正答率、連続学習日数など）
- `WeakCard` - 弱点カード情報（カードID、質問、正答率、復習回数）
- `WeakCardsResponse` - 弱点カード一覧レスポンス
- `ForecastDay` - 日別復習予測（日付、予定カード数）
- `ForecastResponse` - 復習予測レスポンス

**実装ファイル**: `backend/src/models/stats.py`

### 2. StatsService.get_stats() 🔵

**信頼性**: 🔵 *設計文書 architecture.md のサービス層設計より*

DynamoDB からユーザーのカードデータを取得し、以下の統計を集計する:
- 総カード数、学習済みカード数、習得済みカード数
- 全体正答率
- 連続学習日数（ストリーク）
- 今日の復習対象カード数

**実装ファイル**: `backend/src/services/stats_service.py`

### 3. StatsService.get_weak_cards() 🔵

**信頼性**: 🔵 *設計文書 architecture.md のサービス層設計より*

正答率の低いカードを抽出し、弱点カード一覧として返す。正答率でソートし、上位N件を返す。

**実装ファイル**: `backend/src/services/stats_service.py`

### 4. StatsService.get_forecast() 🔵

**信頼性**: 🔵 *設計文書 architecture.md のサービス層設計より*

SRS の next_review_at に基づき、今後7日間の日別復習予定カード数を集計して返す。

**実装ファイル**: `backend/src/services/stats_service.py`

**信頼性**: 🔵 *Pydantic v2 標準パターンより*

- 正常なデータで StatsResponse が生成できること

**信頼性**: 🔵 *設計文書 architecture.md より*
