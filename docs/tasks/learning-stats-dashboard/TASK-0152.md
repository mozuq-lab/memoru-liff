## 主要実装項目

### 1. stats_handler.py エンドポイント実装 🔵

**信頼性**: 🔵 *設計文書 architecture.md のハンドラー設計・既存 handler パターンより*

APIGatewayHttpResolver を使用して以下の3エンドポイントを実装:
- `GET /stats` - 学習統計サマリー取得
- `GET /stats/weak-cards` - 弱点カード一覧取得
- `GET /stats/forecast` - 復習予測取得

各エンドポイントで認証済み user_id を取得し、StatsService の対応メソッドを呼び出す。

**実装ファイル**: `backend/src/api/handlers/stats_handler.py`

### 2. handler.py ルーター登録 🔵

**信頼性**: 🔵 *既存 handler.py のルーター登録パターンより*

既存の handler.py に stats_router を include して、API Gateway からルーティングされるようにする。

**実装ファイル**: `backend/src/api/handler.py`

### 3. エラーハンドリング 🔵

**信頼性**: 🔵 *既存ハンドラーのエラーハンドリングパターンより*

- 認証エラー（401）
- ユーザーデータ未存在時の空レスポンス
- サービス層の例外ハンドリング（500）

**実装ファイル**: `backend/src/api/handlers/stats_handler.py`
