## 主要実装項目

### 1. TypeScript 型定義 🔵

**信頼性**: 🔵 *設計文書 architecture.md のモデル定義・バックエンド Pydantic モデルとの対応より*

バックエンドの Pydantic モデルに対応する TypeScript 型を定義する:
- `StatsResponse` - 学習統計サマリー
- `WeakCard` - 弱点カード情報
- `WeakCardsResponse` - 弱点カード一覧
- `ForecastDay` - 日別復習予測
- `ForecastResponse` - 復習予測レスポンス

**実装ファイル**: `frontend/src/types/stats.ts`

### 2. API サービスメソッド追加 🔵

**信頼性**: 🔵 *既存 api.ts のパターンより*

既存の api.ts に以下のメソッドを追加:
- `getStats()` - GET /stats
- `getWeakCards(limit?: number)` - GET /stats/weak-cards
- `getForecast(days?: number)` - GET /stats/forecast

**実装ファイル**: `frontend/src/services/api.ts`

### 3. useStats カスタムフック 🔵

**信頼性**: 🔵 *既存カスタムフックのパターンより*

統計データを取得・管理するカスタムフックを実装:
- stats, weakCards, forecast の各データ取得
- ローディング状態管理
- エラーハンドリング
- データのリフレッシュ機能

**実装ファイル**: `frontend/src/hooks/useStats.ts`

### 4. 型エクスポート 🔵

**信頼性**: 🔵 *既存 types/index.ts のパターンより*

types/index.ts から stats 型をエクスポートする。

**実装ファイル**: `frontend/src/types/index.ts`（既存ファイルがある場合）

## 基本的なテスト要件

### API サービスメソッドテスト 🔵

**信頼性**: 🔵 *既存 api.ts テストパターンより*

- getStats() が正しい URL で GET リクエストを送ること
- getWeakCards() が limit パラメータを正しく送ること
- getForecast() が days パラメータを正しく送ること
- エラーレスポンス時に例外がスローされること

### useStats フックテスト 🔵

**信頼性**: 🔵 *既存フックテストパターンより*

- 初期状態でローディングが true であること
- データ取得成功時に正しいデータが返ること
- エラー時にエラー状態が設定されること
- リフレッシュ関数でデータが再取得されること
