## タスク概要

`TutorService.get_session()` を改修し、会話履歴を SessionManager 経由で取得できるようにする。DynamoDB バックエンド時は既存の messages フィールドから、AgentCore バックエンド時は SessionManager 経由で履歴を取得する。

## 実装詳細

### 1. get_session の改修 🟡

**信頼性**: 🟡 *設計文書 dataflow.md セッション取得フローから妥当な推測。SessionManager からの会話履歴読み取り方法は実装時に確認が必要*

**実装ファイル**: `backend/src/services/tutor_service.py`

DynamoDB バックエンド時:
- 既存と同じく DynamoDB アイテムの `messages` から取得

AgentCore バックエンド時:
- SessionManager を初期化して Agent にメッセージを復元
- Agent.messages から会話履歴を抽出

### 2. list_sessions は変更なし 🔵

**信頼性**: 🔵 *REQ-006・既存実装より*

`list_sessions()` はメタデータのみ返し、messages は空配列。変更不要。
