## タスク概要

`TutorService.start_session()` をリファクタリングし、AI 挨拶メッセージの生成を SessionManager 付き Agent で行うように変更する。メタデータは引き続き DynamoDB で管理する。

## 実装詳細

### 1. start_session のリファクタリング 🔵

**信頼性**: 🔵 *設計文書 dataflow.md セッション開始フローより*

**実装ファイル**: `backend/src/services/tutor_service.py`

変更後のフロー:
1. デッキ・カードのバリデーション（既存）
2. システムプロンプト構築（既存）
3. `create_tutor_session_manager(session_id, user_id)` で SessionManager 生成
4. `ai_service.generate_response(system_prompt, "挨拶依頼", session_manager=sm)` で挨拶生成
5. 既存アクティブセッション自動終了（既存）
6. メタデータを DynamoDB に保存（messages フィールドは AgentCore バックエンド時は不要）

### 2. メタデータ保存の変更 🔵

**信頼性**: 🔵 *REQ-007・ユーザヒアリングより*

DynamoDB に保存するアイテムから、AgentCore バックエンド時は `messages` フィールドを除外（会話履歴は SessionManager が管理）。DynamoDB バックエンド時は SessionManager が `messages` を管理するため、明示的な保存は不要。

### 3. session_manager_factory の DI 🔵

**信頼性**: 🔵 *設計文書 architecture.md テスタビリティセクションより*

`TutorService.__init__` に `session_manager_factory` パラメータを追加し、テスト時にモック注入可能にする。

**信頼性**: 🔵 *REQ-007 より*

**Then**: `DeckNotFoundError` が発生し、既存のアクティブセッションは消失しない
