## タスク概要

Strands SDK の `SessionManager` インターフェースに準拠した `DynamoDBSessionManager` クラスを実装する。既存の `tutor_sessions` テーブルの `messages` フィールドを利用して会話履歴を管理する。ローカル開発・AgentCore 未対応環境でのフォールバック用バックエンド。

### 2. メッセージ形式変換 🟡

**信頼性**: 🟡 *Strands SDK のメッセージ形式調査から妥当な推測*

Strands Agent のメッセージ形式:
```python
{"role": "user", "content": [{"text": "メッセージ"}]}
```

DynamoDB 保存形式（既存）:
```python
{"role": "user", "content": "メッセージ", "timestamp": "...", "related_cards": [...]}
```

変換ロジックの実装が必要。

### 3. DynamoDB 読み書き 🔵

**信頼性**: 🔵 *既存 tutor_service.py の DynamoDB 操作パターンより*

- `initialize`: `table.get_item(Key={"user_id": ..., "session_id": ...})` で messages を取得
- `sync_agent`: `table.update_item()` で messages を更新

### 4. エンドポイントURL対応 🔵

**信頼性**: 🔵 *既存 tutor_service.py の DynamoDB Local 対応パターンより*

`DYNAMODB_ENDPOINT_URL` / `AWS_ENDPOINT_URL` 環境変数でローカル DynamoDB に接続。
