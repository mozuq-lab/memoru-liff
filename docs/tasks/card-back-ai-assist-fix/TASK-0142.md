## 主要実装項目

### 1. 英語ユーザープロンプトテンプレート追加 🔵

**信頼性**: 🔵 *設計文書 architecture.md「修正 1」・generate.py 既存パターンより*

`refine.py` に英語版の 3 つのヘルパー関数を追加:
- `_get_both_prompt_en(front, back)` — 両方入力時の英語プロンプト
- `_get_front_only_prompt_en(front)` — 表面のみの英語プロンプト
- `_get_back_only_prompt_en(back)` — 裏面のみの英語プロンプト

`get_refine_user_prompt()` 内で `language` パラメータに応じて ja/en を分岐する。

**実装ファイル**: `backend/src/services/prompts/refine.py`

### 2. システムプロンプト関数化 🔵

**信頼性**: 🔵 *設計文書 architecture.md「REFINE_SYSTEM_PROMPT → 関数化」セクションより*

`REFINE_SYSTEM_PROMPT` 定数を `_REFINE_SYSTEM_PROMPT_JA` と `_REFINE_SYSTEM_PROMPT_EN` に分割し、`get_refine_system_prompt(language)` 関数を追加する。

**実装ファイル**: `backend/src/services/prompts/refine.py`

### 3. サービス層の呼び出し元修正 🔵

**信頼性**: 🔵 *設計文書 architecture.md「サービス層への影響」セクションより*

- `bedrock.py:277` — `REFINE_SYSTEM_PROMPT` → `get_refine_system_prompt(language)`
- `strands_service.py:440` — `REFINE_SYSTEM_PROMPT` → `get_refine_system_prompt(language)`
- `prompts/__init__.py` — `REFINE_SYSTEM_PROMPT` を `get_refine_system_prompt` に変更

**実装ファイル**: `backend/src/services/bedrock.py`, `backend/src/services/strands_service.py`, `backend/src/services/prompts/__init__.py`

### 4. テスト追加 🔵

**信頼性**: 🔵 *設計文書 architecture.md「テスト設計」セクション・REQ-TEST-001 より*

`test_refine_prompts.py` に `TestRefinePromptEnglish` クラスを追加:
- `test_en_both_inputs_prompt` — 英語で両方入力のプロンプト生成
- `test_en_front_only_prompt` — 英語で表面のみプロンプト生成
- `test_en_back_only_prompt` — 英語で裏面のみプロンプト生成
- `test_en_system_prompt` — 英語のシステムプロンプト返却

**テストファイル**: `backend/tests/unit/test_refine_prompts.py`
