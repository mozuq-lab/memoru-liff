## 主要実装項目

### 1. UpdateCardRequest拡張 🔵

**信頼性**: 🔵 *設計文書 architecture.md・REQ-101/REQ-102より*

`backend/src/models/card.py` の `UpdateCardRequest` に `interval` フィールドを追加する。

**実装ファイル**: `backend/src/models/card.py`

```python
class UpdateCardRequest(BaseModel):
    front: Optional[str] = Field(None, min_length=1, max_length=1000)
    back: Optional[str] = Field(None, min_length=1, max_length=2000)
    deck_id: Optional[str] = None
    tags: Optional[List[str]] = None
    interval: Optional[int] = Field(None, ge=1, le=365)  # 追加
```

### 2. CardService.update_card拡張 🔵

**信頼性**: 🔵 *設計文書 architecture.md・REQ-003/REQ-004より*

`backend/src/services/card_service.py` の `update_card` メソッドに `interval` パラメータを追加。interval指定時に `next_review_at = now + timedelta(days=interval)` を計算してDynamoDB UpdateExpressionに追加する。

**実装ファイル**: `backend/src/services/card_service.py`

- intervalパラメータを追加
- interval指定時にupdate_partsに `#interval = :interval` と `next_review_at = :next_review_at` を追加
- ease_factor/repetitionsは更新式に含めない
- review_historyには記録しない

### 3. handler.update_card拡張 🔵

**信頼性**: 🔵 *設計文書 architecture.md より*

`backend/src/api/handler.py` の `update_card` ハンドラで `request.interval` を `card_service.update_card()` に渡す。

**実装ファイル**: `backend/src/api/handler.py`

## 注意事項

- DynamoDBのease_factorはstring型で保存されている（float非対応）。interval更新時もこの形式を維持すること
- next_review_atはISO 8601形式（UTC）で保存。`user_id-due-index` GSIのソートキーとしても使用される
- intervalとfront/backを同時に指定することも可能。1つのUpdateExpressionでまとめて更新する
