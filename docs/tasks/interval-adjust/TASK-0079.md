## タスク概要

カード詳細画面（CardDetailPage）にプリセットボタン（1日, 3日, 7日, 14日, 30日）を追加し、ユーザーが復習間隔を手動で調整できるUIを実装する。既存の状態管理パターン（error/successMessage）を再利用する。

## 主要実装項目

### 1. UpdateCardRequest型拡張 🔵

**信頼性**: 🔵 *設計文書 architecture.md フロントエンド変更セクションより*

`frontend/src/types/card.ts` の `UpdateCardRequest` に `interval` フィールドを追加。

**実装ファイル**: `frontend/src/types/card.ts`

```typescript
export interface UpdateCardRequest {
  front?: string;
  back?: string;
  deck_id?: string;
  tags?: string[];
  interval?: number;  // 追加
}
```

### 2. CardDetailPage プリセットボタン追加 🔵

**信頼性**: 🔵 *設計文書 architecture.md・要件定義 REQ-001/REQ-201より*

`frontend/src/pages/CardDetailPage.tsx` のメタ情報セクション下にプリセットボタンを追加。

**実装ファイル**: `frontend/src/pages/CardDetailPage.tsx`

- `isAdjusting` state を追加
- プリセット値: `[1, 3, 7, 14, 30]`
- ボタンタップ → `setIsAdjusting(true)` → `cardsApi.updateCard(id, { interval })` → 成功/失敗処理 → `setIsAdjusting(false)`
- 編集モード中はプリセットボタンを非表示
- 成功時: `setCard(updatedCard)`, `setSuccessMessage('復習間隔を更新しました')`
- 失敗時: `setError('復習間隔の更新に失敗しました')`

**UI構成**:
```
┌─────────────────────────────┐
│ メタ情報                      │
│   次回復習日: 2026-03-07       │
│   復習間隔: 7日                │
├─────────────────────────────┤
│ 復習間隔を調整                 │
│ [1日] [3日] [7日] [14日] [30日] │
└─────────────────────────────┘
```

### 3. ボタン無効化とフィードバック 🟡

**信頼性**: 🟡 *既存UI実装パターン（isSaving）から妥当な推測*

- API呼び出し中: 全プリセットボタンを `disabled` に設定
- タップ領域: 44px以上を確保
- aria-label: 「復習間隔を{N}日に設定」
