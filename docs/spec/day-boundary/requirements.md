# day-boundary 要件定義書（軽量版）

## 概要

復習カードの `next_review_at` がフル datetime で比較されるため、復習した時刻より早い時間にアプリを開くと当日分のカードが表示されない問題を解決する。ユーザーごとに「日付切り替え時刻」（day_start_hour）を設定可能にし、その時刻以降にその日の復習対象カードをまとめて表示する。

## 主要機能要件

**【信頼性レベル凡例】**:
- 🔵 **青信号**: ユーザヒアリング・既存実装を参考にした確実な要件
- 🟡 **黄信号**: 既存実装・設計パターンから妥当な推測による要件
- 🔴 **赤信号**: 推測による要件

### 必須機能（Must Have）

- REQ-001: レビュー送信時、システムは `next_review_at` をユーザーの日付境界時刻に正規化して保存しなければならない 🔵 *ユーザヒアリングより。書き込み時正規化方式を採用*
  - 正規化ロジック: ユーザーの TZ で「有効日付」を計算し（境界前なら前日扱い）、interval 日後の境界時刻を `next_review_at` とする
  - 例 (Asia/Tokyo, day_start_hour=4): 10:00 JST に復習、interval=1 → `next_review_at` = 翌日 04:00 JST
  - 例 (Asia/Tokyo, day_start_hour=4): 01:00 JST に復習（境界前=前日扱い）、interval=1 → `next_review_at` = 当日 04:00 JST

- REQ-002: システムはユーザーごとに `day_start_hour` 設定を保持しなければならない 🔵 *ユーザヒアリングより*
  - デフォルト値: 4（午前4時）
  - 値の範囲: 0〜23（整数）
  - 保存先: `User.settings.day_start_hour`（既存の settings dict パターンを踏襲）

- REQ-003: 設定画面で `day_start_hour` を変更できなければならない 🔵 *ユーザヒアリングより*
  - 選択肢: 0時〜23時（24個）、ドロップダウン（select）形式
  - 夜勤など多様な生活リズムに対応するため全時間帯を選択可能とする
  - 既存の通知時間設定と同じセクション内に配置
  - 保存ボタンで他の設定と一括保存

- REQ-004: カード編集で interval を変更した場合も、`next_review_at` は日付境界時刻に正規化されなければならない 🟡 *REQ-001 との一貫性から妥当な推測*

- REQ-005: 復習対象カードの取得クエリ (`next_review_at <= now`) は変更しない 🔵 *既存実装・書き込み時正規化方式の帰結*

### 基本的な制約

- REQ-401: 既存カードの `next_review_at` はマイグレーション不要。次回復習時に自然に正規化される 🔵 *ユーザヒアリングより*
- REQ-402: 新規カード作成時の `next_review_at = now`（即座に復習可能）は変更しない 🟡 *既存動作の維持が妥当*
- REQ-403: `calculate_sm2()` 関数自体は変更しない。正規化は別関数として実装する 🔵 *既存 SRS アルゴリズムの純粋性を維持*

## 簡易ユーザーストーリー

### ストーリー1: 日付境界時刻による復習カード表示

**私は** 学習者 **として**
**日付の切り替え時刻を設定して、その時刻以降にその日の復習カードをまとめて表示してほしい**
**そうすることで** 深夜に復習しても翌日分が表示されず、自分の生活リズムに合った復習ができる

**関連要件**: REQ-001, REQ-002

### ストーリー2: 日付境界時刻の設定変更

**私は** 学習者 **として**
**設定画面で日付の切り替え時刻を変更したい**
**そうすることで** 自分の生活リズムの変化に合わせて復習タイミングを調整できる

**関連要件**: REQ-003

## 基本的な受け入れ基準

### REQ-001: next_review_at の正規化

**Given**: ユーザーの timezone が "Asia/Tokyo"、day_start_hour が 4
**When**: 10:00 JST にカードを復習し、SM-2 で interval=1 と計算される
**Then**: `next_review_at` が翌日 04:00 JST (前日 19:00 UTC) に設定される

**テストケース**:
- [ ] 正常系: 境界時刻以降の復習で翌日境界に設定される
- [ ] 正常系: 境界時刻以前の復習（前日扱い）で当日境界に設定される
- [ ] 正常系: interval > 1 の場合に正しい日数後の境界に設定される
- [ ] 異常系: timezone が未設定の場合にデフォルト "Asia/Tokyo" が使用される

### REQ-002: day_start_hour 設定の保存

**Given**: ユーザーが設定画面を開いている
**When**: day_start_hour を 4 から 5 に変更して保存する
**Then**: DynamoDB の `settings.day_start_hour` が 5 に更新される

**テストケース**:
- [ ] 正常系: 設定の保存と取得
- [ ] 正常系: デフォルト値 4 の適用（既存ユーザーで未設定の場合）

### REQ-003: 設定画面 UI

**Given**: 設定画面が表示されている
**When**: 日付切り替え時刻の選択肢が表示される
**Then**: 0時〜23時の24個の選択肢がドロップダウンで表示され、現在の設定値が選択状態になる

**テストケース**:
- [ ] 正常系: 選択肢の表示と現在値の反映
- [ ] 正常系: 変更検知と保存ボタンの有効化

## 最小限の非機能要件

- **パフォーマンス**: レビュー送信のレイテンシ増加は最小限（ユーザー設定の取得に1回の DynamoDB GetItem 追加） 🟡
- **後方互換性**: 既存の `settings` dict にフィールド追加のみ。スキーマ変更不要 🔵
