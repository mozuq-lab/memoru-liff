# ローカル開発環境構築 ヒアリング記録

**作成日**: 2026-02-22
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

ローカル開発環境構築のスコープと優先度を明確化するためのヒアリングを実施しました。

## 質問と回答

### Q1: 作業規模の選択

**質問日時**: 2026-02-22
**カテゴリ**: 優先順位
**背景**: ローカル開発環境構築の要件定義をどの詳細度で作成するか決定するため

**回答**: フル機能開発（推奨）

**信頼性への影響**:
- 全要件について詳細な EARS 要件定義、ユーザストーリー、受け入れ基準を作成

---

### Q2: 既存コードベースの詳細分析の要否

**質問日時**: 2026-02-22
**カテゴリ**: 既存設計確認
**背景**: TASK-0048 の実装内容と残タスクは前回セッションで把握済みだが、追加の網羅的調査が必要か確認

**回答**: 不要（推奨）

**信頼性への影響**:
- 既存の実装コードと計画ファイルから十分な情報が得られているため、追加調査なしで要件定義可能
- 信頼性レベルへの影響なし

---

### Q3: dev 環境限定 JWT フォールバックの要件化

**質問日時**: 2026-02-22
**カテゴリ**: 追加要件
**背景**: 計画ファイルに記載されていた機能。SAM local では API Gateway JWT Authorizer が動作しないため、これがないと認証付き API がローカルで動作しない

**回答**: 含める（推奨）

**信頼性への影響**:
- REQ-LD-061〜064 を Must Have として追加（信頼性レベル: 🔵）
- ユーザー承認により計画ファイルの内容が確定

---

### Q4: DynamoDB local SigV4 ハング問題の対応方針

**質問日時**: 2026-02-22
**カテゴリ**: 追加要件
**背景**: 前回セッションで発見された問題。`amazon/dynamodb-local:latest` が boto3 の SigV4 署名付きリクエストでハングする。これにより SAM local API 経由のエンドツーエンドテストが実行不可

**回答**: 要件に含める（推奨）

**信頼性への影響**:
- REQ-LD-071〜073, REQ-LD-103 を Must Have として追加（信頼性レベル: 🔵）
- 原因調査・代替手段（Docker イメージバージョン変更、認証無効化等）の検討を含む

---

### Q5: 追加要件の有無

**質問日時**: 2026-02-22
**カテゴリ**: 追加要件
**背景**: 他にローカル開発環境に関する要件がないか確認

**回答**: ない

**信頼性への影響**:
- スコープ確定。基盤構築 + JWT フォールバック + DynamoDB 問題解決

---

## ヒアリング結果サマリー

### 確認できた事項
- 基盤構築（インポートパス統一、SAM ルーティング修正、DynamoDB 接続設定、Keycloak セットアップ等）はすべて要件として正式に記録
- JWT フォールバックは Must Have として残タスクに含める
- DynamoDB local SigV4 問題は Must Have として残タスクに含める
- 追加要件なし（スコープ確定）

### 追加/変更要件
- REQ-LD-061〜064: JWT フォールバック（計画ファイルから正式要件へ昇格）
- REQ-LD-071〜073, REQ-LD-103: DynamoDB local 接続問題解決（新規発見問題を要件化）

### 残課題
- DynamoDB local SigV4 ハング問題の根本原因が未特定（Docker イメージバージョン、認証設定等の調査が必要）
