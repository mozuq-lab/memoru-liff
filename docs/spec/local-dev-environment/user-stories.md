## エピック1: ローカルインフラ起動

### ストーリー 1.1: ワンコマンドでローカル環境を起動する 🔵

**信頼性**: 🔵 _計画ファイルより_

**私は** 開発者 **として**
**`make local-all` でデータベースと認証サーバーを一括起動したい**
**そうすることで** 開発開始までの手順を最小限にできる

**関連要件**: REQ-LD-041, REQ-LD-042

**詳細シナリオ**:

1. 開発者が `cd backend && make local-all` を実行する
2. DynamoDB local がポート 8000 で起動する
3. DynamoDB Admin がポート 8001 で起動する
4. テーブル（users, cards, reviews）が自動作成される
5. Keycloak がポート 8180 で起動し、realm-local.json がインポートされる
6. すべてのサービスの起動完了を確認できる

**前提条件**:

- Docker / Docker Compose がインストールされている
- ポート 8000, 8001, 8180 が空いている

**優先度**: Must Have

### ストーリー 1.2: Keycloak でテストユーザーにログインする 🔵

**信頼性**: 🔵 _計画ファイル・realm-local.json 設計より_

**私は** 開発者 **として**
**テストユーザー（test-user / test-password-123）で Keycloak にログインしたい**
**そうすることで** LINE アカウントなしで認証フローを確認できる

**関連要件**: REQ-LD-033, REQ-LD-034

**詳細シナリオ**:

1. フロントエンド（localhost:3000）にアクセスする
2. Keycloak のログイン画面にリダイレクトされる
3. test-user / test-password-123 を入力してログインする
4. フロントエンドのホーム画面が表示される
5. ユーザー名「Test User」が画面に表示される

**前提条件**:

- Keycloak が起動している（`make local-keycloak`）
- realm-local.json にテストユーザーが含まれている

**優先度**: Must Have

## エピック2: バックエンド API ローカル実行

### ストーリー 2.1: SAM local で API を起動して動作確認する 🔵

**信頼性**: 🔵 _計画ファイルより_

**私は** 開発者 **として**
**`make local-api` で SAM local API を起動し、全エンドポイントにアクセスしたい**
**そうすることで** デプロイせずにバックエンド API の動作を確認できる

**関連要件**: REQ-LD-011, REQ-LD-012, REQ-LD-043

**詳細シナリオ**:

1. `make local-api` を実行する
2. SAM local API がポート 8080 で起動する
3. API エンドポイント（`/users/me`, `/cards`, `/reviews` 等）にリクエストを送信できる
4. ルーティングが正しく動作し、適切なハンドラーが呼び出される

**前提条件**:

- `make local-all` で DynamoDB local と Keycloak が起動済み
- `make build` で SAM ビルドが完了済み

**優先度**: Must Have

---

### ストーリー 2.2: 認証付き API をローカルで呼び出す 🔵

**信頼性**: 🔵 _計画ファイル・ユーザヒアリングより_

**私は** 開発者 **として**
**Keycloak で取得した JWT トークンを使って認証付き API を呼び出したい**
**そうすることで** 認証フロー込みのエンドツーエンド動作確認ができる

**関連要件**: REQ-LD-061, REQ-LD-062, REQ-LD-063, REQ-LD-064, REQ-LD-101

**詳細シナリオ**:

1. Keycloak でテストユーザーとしてログインし、アクセストークンを取得する
2. `Authorization: Bearer {token}` ヘッダー付きで API にリクエストする
3. ハンドラーが JWT の `sub` クレームからユーザー ID を抽出する
4. ユーザー固有のデータ（カード一覧、設定等）が返却される

**前提条件**:

- SAM local API が起動している
- Keycloak が起動しており、テストユーザーでログイン可能
- `ENVIRONMENT=dev` が env.json で設定されている

**制約事項**:

- JWT フォールバックは dev 環境でのみ有効
- 署名検証は行わない（dev 環境限定のため）

**優先度**: Must Have

---

### ストーリー 2.3: DynamoDB local にデータを読み書きする 🔵

**信頼性**: 🔵 _ユーザヒアリングより_

**私は** 開発者 **として**
**SAM local API 経由で DynamoDB local にデータを読み書きしたい**
**そうすることで** データベース操作を含む完全なエンドツーエンドテストができる

**関連要件**: REQ-LD-021, REQ-LD-022, REQ-LD-071, REQ-LD-072, REQ-LD-073

**詳細シナリオ**:

1. フロントエンドからカード作成リクエストを送信する
2. SAM local Lambda が DynamoDB local にデータを書き込む
3. カード一覧 API でデータが正しく返却される
4. DynamoDB Admin（localhost:8001）でデータを直接確認できる

**前提条件**:

- DynamoDB local が正常に動作している（SigV4 問題が解決済み）
- テーブルが作成済み（setup-tables で自動作成）

**優先度**: Must Have

**備考**: 現在 DynamoDB local の SigV4 ハング問題により未実現。REQ-LD-071〜073 の解決が前提。

## エピック3: フロントエンド連携

### ストーリー 3.1: フロントエンドからローカル API 経由で全画面を操作する 🔵

**信頼性**: 🔵 _計画ファイル・ユーザヒアリングより_

**私は** 開発者 **として**
**フロントエンドの全画面（ホーム、カード一覧、カード作成、設定、LINE連携）が正常に動作することを確認したい**
**そうすることで** リリース前の最終確認がローカルで完結する

**関連要件**: REQ-LD-051, REQ-LD-052

**詳細シナリオ**:

1. `npm run dev` でフロントエンドを起動する
2. Keycloak でログインする
3. ホーム画面で復習カード数が表示される
4. カード一覧でカードが表示される
5. カード作成でテキストを入力して生成できる
6. 設定画面で通知時刻・タイムゾーンを変更できる
7. LINE 連携画面が適切に表示される

**前提条件**:

- `make local-all` と `make local-api` が起動済み
- DynamoDB local が正常動作（SigV4 問題解決済み）
- JWT フォールバックが有効

**優先度**: Must Have
