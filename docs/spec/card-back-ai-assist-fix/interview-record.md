# カード AI 補足機能 レビュー修正 ヒアリング記録

**作成日**: 2026-03-03
**ヒアリング実施**: 要件定義 step4 差分ヒアリング

## ヒアリング目的

コードレビュー（Claude Opus 4.6 + Codex）で発見された指摘事項の修正について、ユーザーと修正スコープ・方針を確認するためのヒアリングを実施しました。

## 質問と回答

### Q1: 修正スコープの確認

**カテゴリ**: 優先順位
**背景**: コードレビューで P1〜P4 の4段階の修正優先度が付与された6件の指摘事項について、今回の修正スコープを確定する必要があった。

**回答**: 以下を選択:
- ✅ P1: language 実装（[High] refine プロンプトの ja/en 分岐実装）
- ✅ P1: body=null 対策（[High] ai_handler.py の型チェック追加）
- ✅ P2: useEffect cleanup（[Low] CardForm のアンマウント時 AbortController.abort() 追加）
- ❌ P3: Bedrock system prompt 分離（スコープ外）

**信頼性への影響**:
- REQ-FIX-001〜005 の信頼性レベルが 🔵 に確定（ユーザーが明示的にスコープ決定）
- P3 のスコープ外決定により、Bedrock system prompt に関する要件は作成不要

---

### Q2: テスト追加の範囲

**カテゴリ**: 追加要件
**背景**: レビューで指摘されたテスト不足箇所について、修正箇所のテストのみか、レビュー指摘の不足ケース全体を対象にするか確認が必要だった。

**回答**: 「不足ケースも含む（推奨）」を選択
- レビューで指摘された不足テスト（Bedrock refine 専用テスト等）も追加対象

**信頼性への影響**:
- REQ-TEST-001〜005 が追加要件として確定（信頼性レベル 🔵）
- テストカバレッジの包括性が向上

---
