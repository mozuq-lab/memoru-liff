## ヒアリング結果サマリー

### 確認できた事項
- AgentCore Memory が利用可能なリージョンにデプロイ予定、SDK も利用可能
- 既存セッションデータの移行は不要（TTL 7日で自然消滅）
- API インターフェースは変更不要（バックエンド内部実装の切り替えのみ）
- 長期メモリ戦略はオプション（今回のスコープ外）
- dev 環境では自動的に dynamodb バックエンドを選択
- tutor_service.py の大幅なリファクタリングは許容

### 追加/変更要件
- REQ-103: dev 環境の自動 dynamodb 選択（新規）
- REQ-404: Memory ID の CDK/SAM 手動作成（新規）
- REQ-405: データ移行不要の明示（新規）
- ~~REQ-501, REQ-502~~: ブランチレビュー指摘のバグ修正 → b6715cd で修正済みのためスコープ外

### 残課題
- AgentCore Memory の具体的な IAM ポリシー定義（SDK ドキュメント参照が必要）
- CDK/SAM での Memory リソース作成方法の詳細（CloudFormation カスタムリソース or CLI）
- DynamoDBSessionManager の Strands SessionManager インターフェース準拠の詳細設計
