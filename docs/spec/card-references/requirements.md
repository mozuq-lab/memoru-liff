# card-references（カード参考情報） 要件定義書

## 概要

カードに参考情報（references）フィールドを追加し、学習内容の出典・参照元を記録できるようにする。各カードに最大5件の参考情報を添付でき、タイプ（URL / 書籍 / メモ）ごとに適切な表示を行う。

## 機能要件（EARS記法）

**【信頼性レベル凡例】**:
- 🔵 **青信号**: 設計文書・ユーザヒアリングを参考にした確実な要件
- 🟡 **黄信号**: 設計文書・ユーザヒアリングから妥当な推測による要件

### 通常要件

- REQ-001: システムは、カードモデルに `references` フィールド（Reference のリスト、デフォルト空リスト）を追加しなければならない 🔵 *機能要件定義より*
- REQ-002: システムは、カード作成時に参考情報を入力できなければならない 🔵 *機能要件定義より*
- REQ-003: システムは、カード編集時に参考情報の追加・編集・削除ができなければならない 🔵 *機能要件定義より*
- REQ-004: システムは、カード詳細画面で参考情報を表示しなければならない（type="url" の場合はクリッカブルリンクとして表示） 🔵 *機能要件定義より*
- REQ-005: システムは、復習画面で裏面表示時に参考情報を表示しなければならない 🔵 *機能要件定義より*

### 制約要件

- REQ-C01: 参考情報は1カードあたり最大5件までとする 🔵 *機能要件定義より*
- REQ-C02: 各参考情報の value は1文字以上500文字以内とする 🔵 *機能要件定義より*

### 非機能要件

- NFR-001: 既存カード（references 未設定）との後方互換性を維持しなければならない。references フィールドが存在しない既存カードは空リストとして扱う 🟡 *DynamoDB スキーマレス特性・既存 tags フィールドの後方互換パターンから妥当な推測*

## データモデル

### Reference モデル

| フィールド | 型 | 説明 |
|-----------|-----|------|
| `type` | `"url"` \| `"book"` \| `"note"` | 参考情報の種別 |
| `value` | string (1-500文字) | 参考情報の内容 |

### type ごとの用途

- **url**: Web URL（例: `https://docs.python.org/3/tutorial/`）。クリッカブルリンクとして表示
- **book**: 書籍参照（例: `入門Python p.42`）。テキストとして表示
- **note**: 自由メモ（例: `授業ノート 第3回`）。テキストとして表示
