## 経緯

ユーザーから学習統計ダッシュボード機能の自律開発を依頼された。具体的な要件のヒアリングは行われておらず、以下を根拠として要件を策定する:

1. **feature-backlog.md**: 機能バックログに記載された学習統計ダッシュボードの仕様案
2. **既存コード分析**: `review_service.py` の `get_review_summary()` が返す統計データ
3. **既存 UI パターン**: HomePage / CardsPage などの既存ページ構成

## 判断基準

- feature-backlog.md に明記されている項目 → 🔵 (確実)
- 既存コードから読み取れる項目 → 🔵 (確実)
- backlog + 既存コードから妥当に推測できる項目 → 🟡 (推測)
- backlog にない追加判断 → 🔴 (仮定)

## 備考

- 時系列グラフ (棒グラフ・折れ線) は backlog に記載があるが、軽量開発の初期フェーズでは必須としない方針
