## REQ-RV-001〜005: 復習セッション基本フロー 🔵

**信頼性**: 🔵 *ユーザヒアリング・既存API仕様より*

### Given（前提条件）
- ユーザーがログイン済み
- 復習対象カードが3枚存在する

### When（実行条件）
- ユーザーが `/review` にアクセスする

### Then（期待結果）
- 復習対象カードが `GET /cards/due` から取得される
- 最初のカードの表面（質問）が表示される
- 進捗バーに「1 / 3」が表示される
- ナビゲーションバーは非表示
- 「戻る」ボタンが表示される

### テストケース

#### 正常系

- [ ] **TC-001-01**: 復習セッション開始 🔵
  - **入力**: `/review` にアクセス（復習対象3枚）
  - **期待結果**: 最初のカードの表面が表示され、進捗「1 / 3」
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-02**: カードフリップ 🔵
  - **入力**: カードエリアをタップ
  - **期待結果**: フリップアニメーションで裏面が表示される
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-03**: フリップ後に採点ボタン表示 🔵
  - **入力**: カードをフリップ済み
  - **期待結果**: 0-5の6個の採点ボタンとスキップボタンが表示される
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-04**: 表面状態では採点ボタン非表示 🟡
  - **入力**: カードが表面表示状態
  - **期待結果**: 採点ボタンが表示されていない
  - **信頼性**: 🟡 *UIデザインから妥当な推測*

- [ ] **TC-001-05**: 進捗バー更新 🔵
  - **入力**: 1枚目を採点後
  - **期待結果**: 進捗バーが「2 / 3」に更新される
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-001-06**: ナビゲーションバー非表示 🔵
  - **入力**: 復習画面表示中
  - **期待結果**: 下部ナビゲーションバーが表示されていない
  - **信頼性**: 🔵 *ユーザヒアリングより*

## REQ-RV-003, REQ-RV-041: 採点とAPI送信 🔵

**信頼性**: 🔵 *ユーザヒアリング・既存API仕様より*

### Given（前提条件）
- カードの裏面が表示されている
- 採点ボタン（0-5）が表示されている

### When（実行条件）
- ユーザーが採点ボタン（grade=4）をタップする

### Then（期待結果）
- `POST /reviews/{card_id}` が `{ "grade": 4 }` で呼び出される
- 次のカードの表面が表示される
- 進捗バーが更新される

### テストケース

#### 正常系

- [ ] **TC-002-01**: grade=0 採点送信 🔵
  - **入力**: 「0」ボタンをタップ
  - **期待結果**: `POST /reviews/{card_id}` に `{ "grade": 0 }` 送信、次のカードへ
  - **信頼性**: 🔵 *既存API仕様より*

- [ ] **TC-002-02**: grade=5 採点送信 🔵
  - **入力**: 「5」ボタンをタップ
  - **期待結果**: `POST /reviews/{card_id}` に `{ "grade": 5 }` 送信、次のカードへ
  - **信頼性**: 🔵 *既存API仕様より*

- [ ] **TC-002-03**: 最後のカード採点後に完了画面 🔵
  - **入力**: 最後のカード（3/3）を採点
  - **期待結果**: 復習完了画面に遷移
  - **信頼性**: 🔵 *ユーザヒアリングより*

#### 異常系

- [ ] **TC-002-E01**: 採点API失敗 🟡
  - **入力**: ネットワークエラー時に採点ボタンをタップ
  - **期待結果**: エラーメッセージ表示、再送信ボタンが表示される
  - **信頼性**: 🟡 *既存実装パターンから推測*

- [ ] **TC-002-E02**: 採点中の二重タップ防止 🟡
  - **入力**: 採点ボタンを連続で素早くタップ
  - **期待結果**: 2回目のタップは無視される（API呼び出しは1回のみ）
  - **信頼性**: 🟡 *UXベストプラクティスから推測*

## REQ-RV-020〜021: スキップ機能 🔵

**信頼性**: 🔵 *ユーザヒアリングより*

### Given（前提条件）
- 復習セッション中にカードが表示されている

### When（実行条件）
- ユーザーが「スキップ」ボタンをタップする

### Then（期待結果）
- APIリクエストは送信されない
- 次のカードの表面が表示される
- 進捗バーが更新される
- スキップされたカードのSRSパラメータは変更されない

### テストケース

#### 正常系

- [ ] **TC-003-01**: カードスキップ 🔵
  - **入力**: 「スキップ」ボタンをタップ
  - **期待結果**: APIリクエストなしで次のカードへ進む
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-003-02**: 全カードスキップ後の完了画面 🟡
  - **入力**: すべてのカード（3枚）をスキップ
  - **期待結果**: 復習完了画面に遷移。復習枚数は0枚と表示
  - **信頼性**: 🟡 *UIロジックから推測*

- [ ] **TC-003-03**: 表面状態からのスキップ 🟡
  - **入力**: カード表面表示状態で「スキップ」をタップ
  - **期待結果**: フリップせずに次のカードへ進む
  - **信頼性**: 🟡 *UIロジックから推測*

## REQ-RV-010〜011: 復習完了画面 🔵

**信頼性**: 🔵 *ユーザヒアリングより*

### Given（前提条件）
- すべての復習対象カードが採点済みまたはスキップ済み

### Then（期待結果）

- [ ] **TC-004-01**: 全カード採点後の完了画面 🔵
  - **入力**: 3枚すべて採点完了
  - **期待結果**: 「復習完了！」メッセージ、「3枚のカードを復習しました」表示
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-004-02**: ホームに戻るボタン 🔵
  - **入力**: 復習完了画面で「ホームに戻る」をタップ
  - **期待結果**: ホーム画面（`/`）に遷移
  - **信頼性**: 🔵 *ユーザヒアリングより*

- [ ] **TC-004-03**: 一部スキップ後の完了画面 🟡
  - **入力**: 3枚中2枚採点、1枚スキップ
  - **期待結果**: 「2枚のカードを復習しました」表示
  - **信頼性**: 🟡 *UIロジックから推測*
