# 復習機能の実装 ヒアリング記録

**作成日**: 2026-02-25
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

既存の設計文書（requirements.md, user-stories.md, api-endpoints.md）および実装を確認した結果、LIFF内の復習画面UIが未定義であることを特定。LINE Webhook経由の復習フロー（ストーリー5.1）は定義済みだが、フロントエンドでの復習体験を明確化するためのヒアリングを実施しました。

## 質問と回答

### Q1: 採点UIの形式

**カテゴリ**: 未定義部分詳細化
**背景**: バックエンドは SM-2 アルゴリズムの 0-5 の 6 段階採点に対応済み。フロントエンドUIとして6段階をそのまま使うか、ユーザビリティのために簡略化するかを確認。

**回答**: **6段階（0-5）ボタン**
- SM-2アルゴリズム準拠。0=全く覚えていない〜5=完璧の6段階ボタンを表示。

**信頼性への影響**:
- REQ-032 (0-5の6段階採点) の UI 実装が明確化 → 信頼性 🔵
- バックエンド API とフロントエンド UI が 1:1 でマッピング

---

### Q2: 復習セッションのフロー

**カテゴリ**: 未定義部分詳細化
**背景**: 復習対象カードの表示方法（1枚ずつ順番 vs リストから選択）を確認。

**回答**: **順番に1枚ずつ**
- 復習対象カードを1枚ずつ表示。表面→裏面フリップ→採点→次のカード。進捗バー付き。

**信頼性への影響**:
- 復習フロー全体の画面遷移が明確化 → 新規要件 REQ-RV-001〜003 追加（信頼性: 🔵）

---

### Q3: 復習画面へのナビゲーション

**カテゴリ**: 未定義部分詳細化
**背景**: 復習画面を新規ページとして作成するか、既存ページに組み込むかを確認。

**回答**: **専用復習ページ（/review）**
- `/review` に新規ページを作成。ホーム画面の「復習を始める」やカード一覧からアクセス。

**信頼性への影響**:
- ルーティング設計が明確化 → REQ-RV-004 追加（信頼性: 🔵）

---

### Q4: 復習完了後の表示

**カテゴリ**: 未定義部分詳細化
**背景**: 復習セッション完了時の画面表示を確認。

**回答**: **シンプルな完了メッセージ**
- 「復習完了！」と復習枚数を表示してホームに戻る。詳細な統計は不要。

**信頼性への影響**:
- 完了画面の仕様が明確化 → REQ-RV-005 追加（信頼性: 🔵）

---

### Q5: スキップ機能

**カテゴリ**: 追加要件
**背景**: 復習中にカードをスキップできるかを確認。

**回答**: **必要**
- スキップボタンでカードを飛ばし、後で復習できる。

**信頼性への影響**:
- スキップ機能要件追加 → REQ-RV-006 追加（信頼性: 🔵）

---

### Q6: カード表示方法

**カテゴリ**: 未定義部分詳細化
**背景**: カードの表面→裏面の切り替え方法を確認。

**回答**: **フリップアニメーション**
- タップでカードが裏返るアニメーション。物理的なカード感覚を再現。

**信頼性への影響**:
- UIアニメーション仕様が明確化 → REQ-RV-007 追加（信頼性: 🔵）

---

### Q7: エントリーポイント

**カテゴリ**: 未定義部分詳細化
**背景**: 復習画面へのアクセス経路を確認。

**回答**: **ホーム「復習を始める」 + カード一覧から**
- ホーム画面の「復習を始める」ボタンを `/review` に変更
- カード一覧の復習対象タブに「復習開始」ボタンを追加

**信頼性への影響**:
- 画面遷移設計が明確化 → REQ-RV-008 追加（信頼性: 🔵）

---

### Q8: 復習中のナビゲーションバー

**カテゴリ**: 未定義部分詳細化
**背景**: 復習画面でナビゲーションバーを表示するかを確認。

**回答**: **非表示**
- 復習に集中させるためナビゲーションバーを非表示。「戻る」ボタンのみで中断可能。

**信頼性への影響**:
- UI レイアウト仕様が明確化 → REQ-RV-009 追加（信頼性: 🔵）

---

## ヒアリング結果サマリー

### 確認できた事項
- 採点UIは SM-2 準拠の6段階（0-5）をそのまま使用
- 1枚ずつ順番に表示するセッション形式
- フリップアニメーションでカードを裏返す
- 専用の `/review` ページを新規作成
- スキップ機能あり
- 復習中はナビゲーションバー非表示
- シンプルな完了メッセージ（枚数のみ）

### 追加/変更要件
- REQ-RV-001〜009: LIFF内復習画面の新規要件 9件追加
- ホーム画面「復習を始める」のリンク先変更: `/cards?tab=due` → `/review`
- カード一覧の復習対象タブに「復習開始」ボタン追加

### 残課題
- 各採点ボタン（0-5）のラベル・色の詳細デザイン（実装時に決定可能）
- フリップアニメーションの具体的な実装方式（CSS transform）
