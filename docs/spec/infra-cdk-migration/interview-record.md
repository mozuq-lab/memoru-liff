# infra-cdk-migration ヒアリング記録

**作成日**: 2026-03-02
**ヒアリング実施**: step4 既存情報ベースの差分ヒアリング

## ヒアリング目的

CloudFormation テンプレートから AWS CDK への移行にあたり、技術選定とスコープを明確化するためのヒアリングを実施しました。

## 質問と回答

### Q1: CDK の言語選定

**カテゴリ**: 既存設計確認
**背景**: フロントエンドが TypeScript、バックエンドが Python のため、どちらの言語で CDK を書くか確認が必要

**回答**: TypeScript

**信頼性への影響**:
- REQ-001 の信頼性レベルが 🔵 に確定

---

### Q2: CDK プロジェクトの配置場所

**カテゴリ**: 既存設計確認
**背景**: 既存の infrastructure/ ディレクトリとの整合性を確認

**回答**: `infrastructure/cdk/`

**信頼性への影響**:
- REQ-002 の信頼性レベルが 🔵 に確定

---

### Q3: CDK 化の対象スタック

**カテゴリ**: 未定義部分詳細化
**背景**: 3つの CloudFormation テンプレートすべてを移行するか確認

**回答**: 全3スタック（Keycloak, LIFF Hosting, Cognito）

**信頼性への影響**:
- REQ-003, REQ-004, REQ-005 の信頼性レベルが 🔵 に確定

---

### Q4: 既存 CloudFormation テンプレートの扱い

**カテゴリ**: 影響範囲
**背景**: CDK 化後に旧テンプレートを残すか削除するか

**回答**: CDK 完成後に削除

**信頼性への影響**:
- REQ-006 の信頼性レベルが 🔵 に確定

---

### Q5: スタック構成

**カテゴリ**: 既存設計確認
**背景**: 3テンプレートを1スタックに統合するか、分離維持するか

**回答**: 3スタック分離（現状と同様）

**信頼性への影響**:
- REQ-002 の構成詳細が 🔵 に確定
