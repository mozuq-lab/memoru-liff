## エピック1: 要約ノート生成

### ストーリー 1.1: デッキの要約ノートを生成する 🔵

**信頼性**: 🔵 *ヒアリングQ1「デッキ＋タグ」・Q2「デッキ詳細画面」・Q4 全項目選択より*

**私は** 学習者 **として**
**デッキ詳細画面から要約ノートを生成したい**
**そうすることで** 復習前にデッキ全体の文脈を把握し、個別カードの暗記効率を上げられる

**関連要件**: REQ-ASN-001, REQ-ASN-011, REQ-ASN-031〜034

**詳細シナリオ**:
1. デッキ詳細画面を開く
2. 「要約ノートを生成」ボタンをタップ
3. AI が処理中のローディング表示を確認
4. 全体像要約・重要ポイント・関連性・学習ヒントを含む要約ノートが表示される
5. 要約ノートを読んでからカードの復習を開始する

**前提条件**:
- 対象デッキに5枚以上のカードが存在する
- ユーザーが認証済みである

**制約事項**:
- 生成は30秒以内に完了すること
- カード数が100枚を超える場合は代表カードが選択される

**優先度**: Must Have

### ストーリー 1.2: タグの要約ノートを生成する 🔵

**信頼性**: 🔵 *ヒアリングQ1「デッキ＋タグ」選択より*

**私は** 学習者 **として**
**特定タグに紐づくカード群の要約ノートを生成したい**
**そうすることで** デッキをまたいだ同一テーマのカードの全体像を把握できる

**関連要件**: REQ-ASN-002, REQ-ASN-301

**詳細シナリオ**:
1. カード一覧でタグフィルターを適用する
2. フィルター結果画面で「要約ノートを生成」ボタンをタップ
3. AI が該当タグのカード群を分析し、要約ノートを生成
4. 要約ノートが表示される

**前提条件**:
- 対象タグに5枚以上のカードが紐づいている
- ユーザーが認証済みである

**優先度**: Should Have

---

### ストーリー 1.3: カードの言語に合わせた要約を受け取る 🔵

**信頼性**: 🔵 *ヒアリングQ7「カードの言語に合わせる」選択より*

**私は** 多言語で学習する学習者 **として**
**カードの言語に合った要約ノートを受け取りたい**
**そうすることで** 学習言語の文脈で全体像を理解できる

**関連要件**: REQ-ASN-051

**詳細シナリオ**:
1. 英語のカードが含まれるデッキの要約を生成する
2. 要約ノートが英語で生成される
3. 日本語のカードが含まれるデッキの要約を生成する
4. 要約ノートが日本語で生成される

**前提条件**:
- デッキ内のカードが特定の言語で記述されている

**優先度**: Should Have

## エピック2: キャッシュと再生成

### ストーリー 2.1: キャッシュされた要約ノートを即座に表示する 🔵

**信頼性**: 🔵 *ヒアリングQ3「キャッシュあり」選択より*

**私は** 学習者 **として**
**以前生成した要約ノートを待ち時間なしで閲覧したい**
**そうすることで** 復習前の準備時間を最小限にできる

**関連要件**: REQ-ASN-021, REQ-ASN-023

**詳細シナリオ**:
1. 以前要約ノートを生成したデッキの詳細画面を開く
2. キャッシュされた要約ノートが即座に表示される（AI生成なし）
3. 500ms以内に表示が完了する

**前提条件**:
- 対象デッキの要約ノートが既にキャッシュに保存されている
- キャッシュが無効化されていない

**優先度**: Must Have

---

### ストーリー 2.2: カード変更後に要約を再生成する 🔵

**信頼性**: 🔵 *ヒアリングQ3「キャッシュあり」選択・キャッシュ無効化ロジックより*

**私は** 学習者 **として**
**カードを追加・変更した後、最新の要約ノートを取得したい**
**そうすることで** 常に正確な全体像を把握できる

**関連要件**: REQ-ASN-022, REQ-ASN-103

**詳細シナリオ**:
1. デッキにカードを追加する
2. デッキ詳細画面を開く
3. 「要約が古くなっています。再生成しますか？」の通知が表示される
4. 「再生成」ボタンをタップ
5. 最新のカード群を反映した要約ノートが生成される

**前提条件**:
- 既存のキャッシュが存在する
- キャッシュ後にカードの追加・更新・削除が行われた

**優先度**: Must Have

## エピック3: エラーハンドリングと制約

### ストーリー 3.1: カード数不足で要約生成を案内される 🔵

**信頼性**: 🔵 *ヒアリングQ5「5枚以上」選択より*

**私は** 学習者 **として**
**カードが少なすぎる場合に適切なフィードバックを受けたい**
**そうすることで** なぜ要約が生成されないか理解し、カードを追加する動機を得る

**関連要件**: REQ-ASN-101

**詳細シナリオ**:
1. カード3枚のデッキ詳細画面を開く
2. 「要約ノートを生成」ボタンが非活性、または「5枚以上のカードが必要です」メッセージが表示される

**前提条件**:
- デッキのカード数が5枚未満

**優先度**: Must Have

---

### ストーリー 3.2: 大規模デッキでも要約を取得する 🔵

**信頼性**: 🔵 *ヒアリングQ6「100枚」選択より*

**私は** 大量のカードを持つ学習者 **として**
**100枚を超えるデッキでも要約ノートを取得したい**
**そうすることで** 大規模な学習内容の全体像を把握できる

**関連要件**: REQ-ASN-102

**詳細シナリオ**:
1. 150枚のカードがあるデッキで要約ノートを生成する
2. システムがSRSパラメータに基づいて代表的な100枚を選択
3. 選択された100枚に基づく要約ノートが生成される

**前提条件**:
- デッキのカード数が100枚を超えている

**優先度**: Should Have

---

### ストーリー 3.3: AI生成エラー時に適切にリカバリーする 🟡

**信頼性**: 🟡 *既存AIエラーハンドリングから妥当な推測*

**私は** 学習者 **として**
**AI生成が失敗した場合にわかりやすいエラーメッセージと再試行手段を得たい**
**そうすることで** 混乱せずにリトライできる

**関連要件**: EDGE-ASN-001

**詳細シナリオ**:
1. 要約ノート生成ボタンをタップ
2. Bedrock APIがタイムアウトする
3. 「要約の生成に失敗しました。しばらく後に再試行してください」のエラーメッセージが表示される
4. 「再試行」ボタンが表示される

**優先度**: Must Have
