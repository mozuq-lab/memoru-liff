## REQ-ASN-001: デッキ単位の要約ノート生成 🔵

**信頼性**: 🔵 *ヒアリングQ1・Q4・既存AIService設計より*

### Given（前提条件）
- ユーザーが認証済みである
- 対象デッキに5枚以上のカードが存在する
- キャッシュが存在しない、または無効化されている

### When（実行条件）
- ユーザーがデッキ詳細画面で「要約ノートを生成」ボタンをタップする

### Then（期待結果）
- AIがデッキ内のカード群を分析し、要約ノートを生成する
- 要約ノートに以下の4セクションが含まれる:
  - 全体像要約
  - 重要ポイントの箇条書き
  - カード間の関連性
  - 学習のヒント
- 生成結果がDynamoDBにキャッシュ保存される
- 要約ノートが画面に表示される

### テストケース

#### 正常系

- [ ] **TC-001-01**: 10枚のカードがあるデッキで要約ノートを生成 🔵
  - **入力**: deck_id（10枚のカード）
  - **期待結果**: 4セクション（全体像・重要ポイント・関連性・ヒント）を含む要約ノートが返却される
  - **信頼性**: 🔵 *ヒアリングQ1・Q4より*

- [ ] **TC-001-02**: 50枚のカードがあるデッキで要約ノートを生成 🔵
  - **入力**: deck_id（50枚のカード）
  - **期待結果**: 全50枚の内容を反映した要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ6より*

- [ ] **TC-001-03**: 英語カードのデッキで英語の要約を生成 🔵
  - **入力**: deck_id（英語カード10枚）
  - **期待結果**: 英語で要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ7より*

- [ ] **TC-001-04**: 日本語カードのデッキで日本語の要約を生成 🔵
  - **入力**: deck_id（日本語カード10枚）
  - **期待結果**: 日本語で要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ7より*

#### 異常系

- [ ] **TC-001-E01**: Bedrock APIタイムアウト時のエラーハンドリング 🟡
  - **入力**: deck_id（Bedrock APIが60秒以上無応答）
  - **期待結果**: AITimeoutError が発生し、「要約の生成に失敗しました。しばらく後に再試行してください」メッセージが返却される
  - **信頼性**: 🟡 *既存AIエラーハンドリングから妥当な推測*

- [ ] **TC-001-E02**: レート制限超過時のエラーハンドリング 🔵
  - **入力**: deck_id（10リクエスト/分の制限を超過）
  - **期待結果**: 429 Too Many Requests が返却される
  - **信頼性**: 🔵 *既存API仕様のレート制限より*

- [ ] **TC-001-E03**: 他ユーザーのデッキへのアクセス拒否 🔵
  - **入力**: 他ユーザーのdeck_id
  - **期待結果**: 403 Forbidden が返却される
  - **信頼性**: 🔵 *既存データ分離設計より*

- [ ] **TC-001-E04**: 存在しないデッキIDの指定 🔵
  - **入力**: 存在しないdeck_id
  - **期待結果**: 404 Not Found が返却される
  - **信頼性**: 🔵 *既存API仕様より*

#### 境界値

- [ ] **TC-001-B01**: カード数が正確に5枚のデッキで要約生成 🔵
  - **入力**: deck_id（5枚のカード）
  - **期待結果**: 要約ノートが正常に生成される
  - **信頼性**: 🔵 *ヒアリングQ5の最小値より*

- [ ] **TC-001-B02**: カード数が4枚のデッキで要約生成拒否 🔵
  - **入力**: deck_id（4枚のカード）
  - **期待結果**: 「5枚以上のカードが必要です」エラーメッセージが返却される
  - **信頼性**: 🔵 *ヒアリングQ5の最小値-1より*

- [ ] **TC-001-B03**: カード数が正確に100枚のデッキで全カード使用 🔵
  - **入力**: deck_id（100枚のカード）
  - **期待結果**: 全100枚を使用して要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ6の上限値より*

- [ ] **TC-001-B04**: カード数が101枚のデッキで代表カード選択 🔵
  - **入力**: deck_id（101枚のカード）
  - **期待結果**: SRSパラメータに基づいて100枚が選択され、要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ6の上限値+1より*

## REQ-ASN-002: タグ単位の要約ノート生成 🔵

**信頼性**: 🔵 *ヒアリングQ1「デッキ＋タグ」選択より*

### Given（前提条件）
- ユーザーが認証済みである
- 対象タグに5枚以上のカードが紐づいている

### When（実行条件）
- ユーザーがタグを指定して要約ノート生成をリクエストする

### Then（期待結果）
- 指定タグに紐づくカード群を分析し、要約ノートが生成される
- デッキをまたいだカードも含まれる
- 結果がキャッシュ保存される

### テストケース

#### 正常系

- [ ] **TC-002-01**: 単一タグで10枚のカードの要約生成 🔵
  - **入力**: tag="英語"（10枚のカード）
  - **期待結果**: 該当タグのカード群の要約ノートが生成される
  - **信頼性**: 🔵 *ヒアリングQ1より*

- [ ] **TC-002-02**: 複数デッキにまたがるタグでの要約生成 🟡
  - **入力**: tag="文法"（デッキA: 5枚, デッキB: 8枚）
  - **期待結果**: 両デッキのカードを統合した要約ノートが生成される
  - **信頼性**: 🟡 *タグの横断性から妥当な推測*

#### 異常系

- [ ] **TC-002-E01**: タグに紐づくカードが4枚以下 🔵
  - **入力**: tag="レアタグ"（3枚のカード）
  - **期待結果**: 「5枚以上のカードが必要です」エラーメッセージ
  - **信頼性**: 🔵 *ヒアリングQ5より*

## REQ-ASN-021: キャッシュ保存と取得 🔵

**信頼性**: 🔵 *ヒアリングQ3「キャッシュあり」選択より*

### Given（前提条件）
- 要約ノートが正常に生成された

### When（実行条件）
- 生成が完了した時

### Then（期待結果）
- 生成結果がDynamoDBに保存される
- 次回アクセス時にAI生成をスキップしてキャッシュから返却される

### テストケース

#### 正常系

- [ ] **TC-021-01**: 生成後のキャッシュ保存 🔵
  - **入力**: 要約ノート生成成功
  - **期待結果**: DynamoDBにsource_type, source_id, content, generated_at等が保存される
  - **信頼性**: 🔵 *ヒアリングQ3より*

- [ ] **TC-021-02**: キャッシュからの即座取得 🔵
  - **入力**: キャッシュ済みデッキの要約ノート取得リクエスト
  - **期待結果**: 500ms以内にキャッシュされた要約ノートが返却される
  - **信頼性**: 🔵 *ヒアリングQ3より*

## REQ-ASN-022: キャッシュ無効化 🔵

**信頼性**: 🔵 *ヒアリングQ3「キャッシュあり」選択・無効化ロジックより*

### Given（前提条件）
- デッキの要約ノートがキャッシュ済みである

### When（実行条件）
- 対象デッキのカードが追加、更新、または削除された時

### Then（期待結果）
- キャッシュが無効化される
- 次回アクセス時に「要約が古くなっています。再生成しますか？」の通知が表示される

### テストケース

#### 正常系

- [ ] **TC-022-01**: カード追加時のキャッシュ無効化 🔵
  - **入力**: キャッシュ済みデッキに新カード追加
  - **期待結果**: キャッシュのステータスが invalidated に更新される
  - **信頼性**: 🔵 *ヒアリングQ3より*

- [ ] **TC-022-02**: カード更新時のキャッシュ無効化 🔵
  - **入力**: キャッシュ済みデッキのカードfront/backを変更
  - **期待結果**: キャッシュのステータスが invalidated に更新される
  - **信頼性**: 🔵 *ヒアリングQ3より*

- [ ] **TC-022-03**: カード削除時のキャッシュ無効化 🔵
  - **入力**: キャッシュ済みデッキからカードを削除
  - **期待結果**: キャッシュのステータスが invalidated に更新される
  - **信頼性**: 🔵 *ヒアリングQ3より*

- [ ] **TC-022-04**: 無効化後の再生成通知表示 🟡
  - **入力**: 無効化済みデッキの詳細画面を開く
  - **期待結果**: 「要約が古くなっています。再生成しますか？」が表示される
  - **信頼性**: 🟡 *UX設計として妥当な推測*
