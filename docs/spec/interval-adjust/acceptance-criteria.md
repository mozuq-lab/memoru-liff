## REQ-001: プリセットボタン表示 🔵

**信頼性**: 🔵 *ユーザヒアリング UI形式・プリセット値より*

### Given（前提条件）
- ユーザーがログイン済みである
- カードが存在する
- カード詳細画面を表示している（表示モード）

### When（実行条件）
- カード詳細画面のメタ情報セクションを確認する

### Then（期待結果）
- 「1日」「3日」「7日」「14日」「30日」の5つのプリセットボタンが表示される
- 各ボタンのタップ領域は44px以上である

### テストケース

#### 正常系

- [ ] **TC-001-01**: カード詳細画面にプリセットボタン5つが表示されること 🔵
  - **入力**: 任意のカードの詳細画面を開く
  - **期待結果**: 「1日」「3日」「7日」「14日」「30日」のボタンが表示される

- [ ] **TC-001-02**: 編集モード時にプリセットボタンが非表示であること 🟡
  - **入力**: カード詳細画面で編集ボタンを押す
  - **期待結果**: プリセットボタンが表示されない
  - **信頼性**: 🟡 *既存の編集モード切替パターンから妥当な推測*

## REQ-002: interval更新API呼び出し 🔵

**信頼性**: 🔵 *ユーザヒアリング API設計より*

### Given（前提条件）
- ユーザーがログイン済みである
- カード詳細画面を表示している

### When（実行条件）
- プリセットボタン（例: 「7日」）をタップする

### Then（期待結果）
- `PUT /cards/:card_id` APIが `{ "interval": 7 }` で呼び出される
- APIレスポンスのintervalが7である
- カード詳細画面のメタ情報が更新される

### テストケース

#### 正常系

- [ ] **TC-002-01**: プリセットボタン「1日」タップでinterval=1に更新されること 🔵
  - **入力**: 「1日」ボタンをタップ
  - **期待結果**: カードのintervalが1に更新される

- [ ] **TC-002-02**: プリセットボタン「30日」タップでinterval=30に更新されること 🔵
  - **入力**: 「30日」ボタンをタップ
  - **期待結果**: カードのintervalが30に更新される

- [ ] **TC-002-03**: interval更新後にカード詳細のメタ情報が即座に更新されること 🔵
  - **入力**: 任意のプリセットボタンをタップ
  - **期待結果**: 「復習間隔」と「次回復習日」が新しい値に更新される

## REQ-101/102: バリデーション 🔵

**信頼性**: 🔵 *ユーザヒアリング 値の制約より*

### Given（前提条件）
- ユーザーがログイン済みである

### When（実行条件）
- 不正なinterval値でAPIを呼び出す

### Then（期待結果）
- 400 Bad Requestが返される
- エラーメッセージが含まれる

### テストケース

#### 異常系

- [ ] **TC-101-01**: interval=0で400エラーが返ること 🔵
  - **入力**: `PUT /cards/:id` に `{ "interval": 0 }` を送信
  - **期待結果**: 400 Bad Request

- [ ] **TC-101-02**: interval=-1で400エラーが返ること 🔵
  - **入力**: `PUT /cards/:id` に `{ "interval": -1 }` を送信
  - **期待結果**: 400 Bad Request

- [ ] **TC-102-01**: interval=366で400エラーが返ること 🔵
  - **入力**: `PUT /cards/:id` に `{ "interval": 366 }` を送信
  - **期待結果**: 400 Bad Request

#### 境界値

- [ ] **TC-101-B01**: interval=1で正常に更新されること 🔵
  - **入力**: `PUT /cards/:id` に `{ "interval": 1 }` を送信
  - **期待結果**: 200 OK, intervalが1に更新

- [ ] **TC-102-B01**: interval=365で正常に更新されること 🔵
  - **入力**: `PUT /cards/:id` に `{ "interval": 365 }` を送信
  - **期待結果**: 200 OK, intervalが365に更新
