## エピック1: 復習間隔の手動調整

### ストーリー 1.1: プリセットボタンで間隔を短縮する 🔵

**信頼性**: 🔵 *ユーザヒアリング UI形式・プリセット値・SM-2説明での合意より*

**私は** 学習者 **として**
**覚えが悪いカードの復習間隔を短くしたい**
**そうすることで** そのカードをより頻繁に復習して記憶を定着させられる

**関連要件**: REQ-001, REQ-002, REQ-003, REQ-004

**詳細シナリオ**:
1. カード一覧からカードをタップしてカード詳細画面を開く
2. 現在の復習間隔（例: 14日）と次回復習日を確認する
3. プリセットボタン「3日」をタップする
4. 復習間隔が3日に更新され、次回復習日が「現在日時+3日」に再計算される
5. 成功メッセージが表示される

**前提条件**:
- ユーザーがログイン済みであること
- カードが存在すること

**優先度**: Must Have

---

### ストーリー 1.2: プリセットボタンで間隔を延長する 🔵

**信頼性**: 🔵 *ユーザヒアリング UI形式・プリセット値より*

**私は** 学習者 **として**
**すでに覚えたカードの復習間隔を長くしたい**
**そうすることで** 他の覚えていないカードの復習に集中できる

**関連要件**: REQ-001, REQ-002, REQ-003, REQ-004

**詳細シナリオ**:
1. カード詳細画面で現在の復習間隔（例: 1日）を確認する
2. プリセットボタン「30日」をタップする
3. 復習間隔が30日に更新され、次回復習日が「現在日時+30日」に再計算される
4. ease_factorとrepetitionsは変更されない

**前提条件**:
- ユーザーがログイン済みであること
- カードが存在すること

**優先度**: Must Have

---

### ストーリー 1.3: 間隔調整でエラーが発生する 🟡

**信頼性**: 🟡 *既存エラーハンドリングパターンから妥当な推測*

**私は** 学習者 **として**
**間隔調整が失敗した場合にエラー内容を知りたい**
**そうすることで** 問題を理解し、再度試行できる

**関連要件**: REQ-103, EDGE-001

**詳細シナリオ**:
1. カード詳細画面でプリセットボタンをタップする
2. ネットワークエラーなどでAPI呼び出しが失敗する
3. エラーメッセージが表示される
4. カードの復習間隔・次回復習日は変更前の値のまま

**前提条件**:
- ユーザーがログイン済みであること

**優先度**: Must Have
