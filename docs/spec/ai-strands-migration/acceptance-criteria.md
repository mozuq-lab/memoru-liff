## REQ-SM-001: Strands Agents SDK 統合 🔵

**信頼性**: 🔵 *ユーザヒアリング・Strands Agents SDK 仕様より*

### Given（前提条件）
- Strands Agents SDK が `requirements.txt` にインストールされている
- Bedrock プロバイダーが設定可能

### When（実行条件）
- AI サービスが初期化される

### Then（期待結果）
- Strands Agent インスタンスが Bedrock プロバイダーで作成される
- 環境変数 `BEDROCK_MODEL_ID` でモデルが切り替え可能

### テストケース

#### 正常系

- [ ] **TC-001-01**: Strands Agent の初期化が正常に行われる 🔵
  - **入力**: model_id="anthropic.claude-3-haiku-20240307-v1:0"
  - **期待結果**: Agent instance が Bedrock プロバイダーで作成される
  - **信頼性**: 🔵 *Strands Agents SDK の基本的な初期化パターン*

- [ ] **TC-001-02**: 環境変数でモデル ID を切り替えられる 🔵
  - **入力**: `BEDROCK_MODEL_ID` 環境変数に別モデルを設定
  - **期待結果**: 指定されたモデルで Agent が作成される
  - **信頼性**: 🔵 *既存実装 `bedrock.py` の `os.environ.get("BEDROCK_MODEL_ID")` パターンより*

#### 異常系

- [ ] **TC-001-E01**: 無効なモデル ID でエラーが返される 🟡
  - **入力**: 無効なモデル ID 文字列
  - **期待結果**: 適切なエラー（BedrockServiceError 相当）が発生
  - **信頼性**: 🟡 *Strands Agents のエラーハンドリングパターンから推測*

## REQ-SM-002: カード生成機能の移行 🔵

**信頼性**: 🔵 *既存実装 `bedrock.py` + ユーザヒアリングより*

### Given（前提条件）
- `USE_STRANDS=true` が設定されている
- Bedrock API へのアクセスが可能

### When（実行条件）
- `POST /cards/generate` にリクエストが送信される

### Then（期待結果）
- Strands Agents 経由でカードが生成される
- 既存の `GenerateCardsResponse` 形式でレスポンスが返される

### テストケース

#### 正常系

- [ ] **TC-002-01**: Strands Agents 経由でカード生成が正常動作する 🔵
  - **入力**: `{text: "テスト文章", card_count: 3, difficulty: "medium", language: "ja"}`
  - **期待結果**: `GenerationResult` に 3 枚のカード、`model_used`、`processing_time_ms` が含まれる
  - **信頼性**: 🔵 *既存テスト `test_bedrock.py` の正常系パターンより*

- [ ] **TC-002-02**: 既存の GenerationResult 形式が維持される 🔵
  - **入力**: 有効なカード生成リクエスト
  - **期待結果**: レスポンスに `cards`, `input_length`, `model_used`, `processing_time_ms` フィールドが存在
  - **信頼性**: 🔵 *既存実装 `GenerationResult` dataclass より*

- [ ] **TC-002-03**: 各難易度レベルで正常動作する 🔵
  - **入力**: difficulty = "easy" / "medium" / "hard"
  - **期待結果**: 各難易度に応じたカードが生成される
  - **信頼性**: 🔵 *既存テスト `TestPrompts` クラスより*

- [ ] **TC-002-04**: 日本語・英語両方で正常動作する 🔵
  - **入力**: language = "ja" / "en"
  - **期待結果**: 指定言語でカードが生成される
  - **信頼性**: 🔵 *既存実装 `prompts.py` の言語対応より*

#### 異常系

- [ ] **TC-002-E01**: タイムアウト時に 504 が返される 🔵
  - **入力**: タイムアウトが発生する API 呼び出し
  - **期待結果**: 504 Gateway Timeout とエラーメッセージ
  - **信頼性**: 🔵 *既存実装 `handler.py` の `BedrockTimeoutError → 504` マッピングより*

- [ ] **TC-002-E02**: レート制限時に 429 が返される 🔵
  - **入力**: レート制限中の API 呼び出し
  - **期待結果**: 429 Too Many Requests
  - **信頼性**: 🔵 *既存実装 `handler.py` の `BedrockRateLimitError → 429` マッピングより*

- [ ] **TC-002-E03**: レスポンス解析失敗時に 500 が返される 🔵
  - **入力**: 不正な JSON 形式のモデルレスポンス
  - **期待結果**: 500 Internal Server Error
  - **信頼性**: 🔵 *既存実装 `handler.py` の `BedrockParseError → 500` マッピングより*

## REQ-SM-003: 回答採点・AI評価 🔵

**信頼性**: 🔵 *ユーザヒアリング「回答採点・AI評価」選択より*

### Given（前提条件）
- ユーザーが復習画面でカードの問題に回答している
- カードに front（問題）と back（正解）が設定されている

### When（実行条件）
- AI 採点リクエストが送信される

### Then（期待結果）
- AI が回答を分析し、0-5 の SRS グレードと採点理由を返す

### テストケース

#### 正常系

- [ ] **TC-003-01**: 正答に対して高グレード（4-5）が返される 🟡
  - **入力**: `{card_front: "日本の首都は？", card_back: "東京", user_answer: "東京"}`
  - **期待結果**: grade 4 or 5 + 肯定的なフィードバック
  - **信頼性**: 🟡 *AI 採点の具体的な挙動は設計フェーズで詳細化*

- [ ] **TC-003-02**: 部分正答に対して中グレード（2-3）が返される 🟡
  - **入力**: `{card_front: "光合成の過程を説明せよ", card_back: "...", user_answer: "部分的な回答"}`
  - **期待結果**: grade 2 or 3 + 具体的な改善フィードバック
  - **信頼性**: 🟡 *AI 採点のグレーディング基準は設計フェーズで詳細化*

- [ ] **TC-003-03**: 誤答に対して低グレード（0-1）が返される 🟡
  - **入力**: `{card_front: "日本の首都は？", card_back: "東京", user_answer: "大阪"}`
  - **期待結果**: grade 0 or 1 + 正解の提示
  - **信頼性**: 🟡 *AI 採点のグレーディング基準は設計フェーズで詳細化*

#### 異常系

- [ ] **TC-003-E01**: 空の回答に対して 400 エラーが返される 🔵
  - **入力**: `user_answer` が空文字
  - **期待結果**: 400 Bad Request
  - **信頼性**: 🔵 *既存のバリデーションパターン（Pydantic v2）より*

- [ ] **TC-003-E02**: 存在しないカード ID に対して 404 が返される 🔵
  - **入力**: 無効な card_id
  - **期待結果**: 404 Not Found
  - **信頼性**: 🔵 *既存の API エラーハンドリングパターンより*

#### 境界値

- [ ] **TC-003-B01**: 非常に長い回答（2000文字以上）に対して正常処理される 🟡
  - **入力**: 2000 文字以上の `user_answer`
  - **期待結果**: タイムアウト内で処理完了
  - **信頼性**: 🟡 *既存の入力バリデーション（10-2000 chars）パターンから推測*
