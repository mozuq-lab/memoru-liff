# Memoru LIFF 先進機能アイデア集

作成日: 2026-03-05

## 概要

SRS（間隔反復）+ LINE + AI 基盤を活かした、次世代の記憶学習体験を実現するアイデア集。
既存のバックログ（`feature-backlog.md`）とは別に、中長期的な差別化機能の候補をまとめる。

---

## 評価基準

| 実装コスト | 目安 |
|-----------|------|
| S（小）   | 1〜3日。既存コンポーネント流用、インフラ変更なし |
| M（中）   | 1〜2週間。新規 API / コンポーネント追加、軽微なインフラ変更 |
| L（大）   | 2〜4週間。新規サービス層、外部連携、インフラ構築が必要 |

## 1. 忘却カジノ（Forgetting Curve Betting）

### 概要
復習前に「このカード、覚えていると思うか？」にポイントを賭ける。自分の記憶に対する予測精度（メタ認知能力）を可視化・強化するゲーミフィケーション。

### 仕組み
- 復習カード提示時に「賭ける / 賭けない」を選択
- 賭け額を設定（持ちポイントから）
- 正解 → ポイント倍増、不正解 → 没収
- 自信過剰（いつも賭けて間違える）や過小評価（いつも賭けずに正解）をAIが検出
- LINE 友達とポイントランキングで競争

### 実装コスト: M（中）

| 項目 | 内容 |
|------|------|
| バックエンド | ポイントシステム（users テーブルに `points` 属性追加）、賭け履歴テーブル、メタ認知分析ロジック |
| フロントエンド | ReviewPage に賭け UI 追加、ポイント表示、ランキング画面 |
| インフラ | DynamoDB テーブル追加（賭け履歴）のみ。外部サービス不要 |

### 見解
**おすすめ度: ★★★★☆**
実装コストに対するインパクトが大きい。復習のモチベーション維持が SRS アプリ最大の課題であり、それを直接解決する。メタ認知の可視化は学術的にも価値がある。ただし、ポイント経済のバランス調整に継続的なチューニングが必要。

## 2. 記憶分解 AI（Memory Decomposition）

### 概要
何度も忘れるカードを AI が「なぜ覚えられないか」を診断し、前提知識カード・比較カード・具体例カードに自動分解する。

### 仕組み
- SRS で繰り返し Lapse するカードを自動検出（例: 3回以上連続不正解）
- AI が原因を分析（前提知識の欠如 / 似た概念との混同 / 抽象度が高すぎ）
- 原因に応じた補助カードを自動生成
- 補助カードをマスターすると元カードの復習に進む「スキルツリー」的依存関係

### 実装コスト: L（大）

| 項目 | 内容 |
|------|------|
| バックエンド | Lapse 検出ロジック（CardService 拡張）、AI 分析プロンプト設計、カード依存関係モデル、自動生成パイプライン |
| フロントエンド | スキルツリー UI、分解提案通知、依存関係表示 |
| AI/Bedrock | 診断プロンプトの設計・チューニングが核心。既存の generate パイプラインを拡張 |
| インフラ | cards テーブルに `parent_card_id` / `dependency_type` 追加 |

### 見解
**おすすめ度: ★★★★★**
最も推奨。「なぜ覚えられないか」に踏み込む SRS アプリは市場に存在せず、明確な差別化要因になる。既存の Bedrock Claude 統合・カード生成パイプラインとの親和性が非常に高い。AI プロンプト設計が成功の鍵だが、段階的にリリース可能（まず手動トリガー → 自動検出）。

## 3. ゴースト復習（Ghost Review）

### 概要
過去の自分の復習パフォーマンス（速度・正誤）を「ゴースト」として記録し、今日の復習時に並走させる。マリオカートのタイムアタックのように、自分の成長を体感する。

### 仕組み
- 復習セッションの各カードの回答時間・正誤を記録（既存の reviews テーブルを活用）
- 復習時に「1週間前の自分」のゴーストデータを取得
- カードごとに「過去の自分: 12秒 → 今の自分: 3秒」をリアルタイム表示
- セッション終了時に勝敗判定と成長サマリー
- LINE 通知:「昨日の自分に勝ちました！」

### 実装コスト: S〜M（小〜中）

| 項目 | 内容 |
|------|------|
| バックエンド | reviews テーブルに `response_time_ms` 追加（既存 API の軽微な拡張）、ゴーストデータ取得 API |
| フロントエンド | ReviewPage にゴースト表示 UI（プログレスバー比較）、セッション結果画面 |
| インフラ | 変更なし。既存テーブル・インフラで完結 |

### 見解
**おすすめ度: ★★★★☆**
実装コストが低いのに体験の変化が大きい。「成長の実感」は継続率に直結する。reviews テーブルに回答時間を追加するだけでバックエンドはほぼ完成。フロントエンドの演出次第で非常に楽しい機能になる。早期実装候補。

## 4. 記憶の地図（Memory Cartography）

### 概要
学習した知識を AI が意味解析し、概念間の関係をナレッジグラフとして自動可視化。「知識の領土」を広げていく探索的な学習体験。

### 仕組み
- カードの内容を AI がベクトル化（埋め込み）し、概念間の意味的距離を計算
- 「島」= 知識クラスター、「橋」= 概念間のつながり、「霧」= 未学習領域
- 復習で定着するとエリアが明るくなり、Lapse すると霧が戻る
- AI が「この2つの島を橋でつなぐカードを作りませんか？」と知識の穴を提案

### 実装コスト: XL（特大）

| 項目 | 内容 |
|------|------|
| バックエンド | テキスト埋め込み（Bedrock Titan Embeddings or 外部 API）、クラスタリングアルゴリズム、グラフデータ構造 |
| フロントエンド | インタラクティブなグラフ可視化（D3.js / vis.js / React Flow 等）、ズーム・パン操作 |
| AI/Bedrock | 埋め込みモデル利用、知識の穴検出ロジック |
| インフラ | 埋め込みベクトル保存（DynamoDB or 専用ベクトル DB）、バッチ処理 |

### 見解
**おすすめ度: ★★★☆☆**
ビジョンとしては最も壮大で魅力的だが、実装コストが非常に高い。グラフ可視化の UX をモバイル（LIFF）で快適にするのが技術的に難しい。MVP としてシンプルなクラスターマップから始め、段階的に拡張する戦略が現実的。長期ロードマップ向き。

## 5. 逆 SRS — 教えて定着（Teach to Learn）

### 概要
AI が「わからないフリ」をして生徒役になり、ユーザーが先生として概念を説明する。ファインマン・テクニックを SRS に統合した新しい復習形式。

### 仕組み
- 通常復習の代わりに「先生モード」を選択可能
- AI が生徒として質問:「先生、○○って△△のことですか？」（わざと間違える）
- ユーザーが間違いを指摘・正しく説明できれば高スコア
- AI の間違い方はよくある誤解を反映（プロンプトで制御）
- ヒントなしで説明できた = 完全定着、説明できない = 理解が浅い
- LINE チャット形式で自然に実行

### 実装コスト: M〜L（中〜大）

| 項目 | 内容 |
|------|------|
| バックエンド | AI 対話セッション管理、「わざと間違える」プロンプト設計、回答評価ロジック |
| フロントエンド | チャット UI（LINE 風）を ReviewPage に統合、先生モード切替 |
| AI/Bedrock | プロンプトエンジニアリングが核心。カードの内容から「もっともらしい間違い」を生成する品質が鍵 |
| インフラ | 対話履歴保存（DynamoDB）。大きなインフラ変更は不要 |

### 見解
**おすすめ度: ★★★★★**
最も推奨するアイデアの一つ。LINE チャット UI との親和性が完璧で、ファインマン・テクニック × SRS の組み合わせは市場に前例がない。Bedrock Claude の強みを最大限に活かせる。実装の核心は「賢くわざと間違える」プロンプト設計だが、これは段階的に改善可能。まず一部のカードで実験的に導入し、効果を測定するのが良い。

## 6. 感情エコー（Emotional Echo）

### 概要
復習時の感情を 1 タップで記録し、感情と記憶定着の相関を分析。個人の「感情 × 記憶」パターンを発見し、感情的に最適な復習体験を構築する。

### 仕組み
- 各カード復習後に感情タグを 1 タップ（😤 😐 😮 😊 🔥）
- データ蓄積後、AI が「あなたは驚いた時の記憶が最も定着する」等のパーソナル分析
- 復習セッション中に感情の単調化（退屈→集中力低下）を検出し、カード提示順を動的に変更
- 長期的に「自分だけの最適な感情カーブ」を可視化

### 実装コスト: M（中）

| 項目 | 内容 |
|------|------|
| バックエンド | reviews テーブルに `emotion_tag` 追加、感情-定着率の相関分析ロジック |
| フロントエンド | ReviewPage に感情タグ UI（1タップ）、統計画面に感情分析セクション |
| AI/Bedrock | 相関分析は統計処理で可能。AI は分析結果の自然言語化に使用 |
| インフラ | 変更なし |

### 見解
**おすすめ度: ★★★☆☆**
1タップの手軽さが良く、データ収集コストが低い。ただし、感情と記憶定着の相関が統計的に有意になるまでに大量のデータが必要で、ユーザーが価値を感じるまでに時間がかかる。初期は単純な感情ログとして実装し、データが溜まってから分析機能を追加する段階的アプローチが現実的。

## 7. タイムカプセル復習（Time Capsule Review）

### 概要
カード作成時に「未来の自分へのメモ」を記録。数ヶ月後の復習時にそのメモが再生され、過去の学習コンテキストが蘇る。

### 仕組み
- カード作成 / 復習時に「未来の自分へのメモ」をテキストまたは音声で記録
- メモには「このカードのコツ」「つまずいたポイント」「関連付けた記憶」等を記録
- 設定期間後（例: 3ヶ月後）の復習時にメモが表示 / 再生
- うまくいった学習戦略が自然にアーカイブされる

### 実装コスト: S（小）

| 項目 | 内容 |
|------|------|
| バックエンド | cards テーブルに `memo` / `memo_reveal_after_days` 属性追加、復習 API でメモ表示判定 |
| フロントエンド | CardForm にメモ入力欄追加、ReviewPage にメモ表示 UI |
| インフラ | 変更なし（テキストメモの場合）。音声メモは S3 が必要 |

### 見解
**おすすめ度: ★★★★☆**
実装コストが最も低いのに、ユーザー体験の深みが大きく増す。テキストメモのみなら 1〜2 日で実装可能。音声メモは S3 依存だが後回しにできる。既存の CardForm / ReviewPage への軽微な拡張で済み、リスクが非常に低い。即座に実装できる候補。

## 8. 夢の中の復習（Hypnagogic Review）

### 概要
睡眠直前の「入眠時」に特化した超短時間マイクロ復習。睡眠中の記憶固定（memory consolidation）を最大化する。

### 仕組み
- Apple Health / Google Fit 連携で就寝時間を学習（または手動設定）
- 就寝 5 分前に LINE 通知:「おやすみ復習 — 3 枚だけ」
- カードは画像・音声のみ（テキストを読ませない設計）で入眠時の脳に負荷をかけない
- その日最も Lapse したカードを厳選
- 翌朝の復習で効果を測定し、「おやすみ復習」の有効性をパーソナルに検証

### 実装コスト: M〜L（中〜大）

| 項目 | 内容 |
|------|------|
| バックエンド | 就寝時間管理、マイクロ復習セッション生成、効果測定ロジック |
| フロントエンド | マイクロ復習専用 UI（ダークモード、最小限の操作）、設定画面 |
| LINE 連携 | LINE Messaging API でスケジュール通知（EventBridge + Lambda） |
| インフラ | EventBridge スケジューラー、ユーザー別通知時間管理 |

### 見解
**おすすめ度: ★★★☆☆**
コンセプトは非常にユニークだが、Apple Health / Google Fit 連携は LIFF（ブラウザベース）からは直接アクセスできず、手動の就寝時間設定が現実的。LINE 通知のスケジューリング自体は既存の LINE Messaging API で実装可能。効果の科学的根拠はあるが、ユーザーに効果を実感させるまでのハードルが高い。

## 9. 記憶の DNA（Memory DNA）

### 概要
学習パターンを多次元ベクトル化し、「記憶の DNA」として可視化。似た DNA の学習者をマッチングし、学習戦略を交換・最適化する。

### 仕組み
- 学習速度、忘却パターン、得意ジャンル、最適復習時間帯等を多次元ベクトル化
- 二重螺旋ビジュアルで「あなたの記憶 DNA」を生成
- 似た DNA の学習者を自動マッチング →「この人のデッキが合うかも」
- DNA 同士を「交配」→ 2 人の学習パターンの良いとこ取りしたハイブリッド復習スケジュール生成

### 実装コスト: XL（特大）

| 項目 | 内容 |
|------|------|
| バックエンド | 学習パターン分析エンジン、ベクトル化、マッチングアルゴリズム、スケジュール最適化 |
| フロントエンド | DNA ビジュアライゼーション（Canvas / WebGL）、マッチング UI、プロフィール画面 |
| インフラ | ユーザー間マッチング基盤、プライバシー管理、ソーシャル機能基盤 |

### 見解
**おすすめ度: ★★☆☆☆**
コンセプトは面白いが、実装コストが非常に高く、十分なユーザー数がないとマッチング機能が成立しない。ソーシャル機能基盤やプライバシー管理も大きな課題。ユーザーベースが成長した後の長期構想として温存すべき。

## 10. 記憶の天気予報（Memory Weather Forecast）

### 概要
SRS データと学習履歴から「今日の忘却リスク」を天気予報のメタファーで毎朝通知。復習を「義務」から「天気に備える行動」に変換する。

### 仕組み
- 各カードの忘却確率を SRS アルゴリズムから計算（既存データで可能）
- 忘却リスクを天気に変換:「☀️ 安定」「☁️ やや不安」「☔ 危険」「🌪️ 台風警報」
- 毎朝 LINE 通知:「今日の記憶天気: ☁️ — 英単語に注意報。夕方までに 5 枚推奨」
- 週間予報:「来週水曜に ☔ — 32 枚が危険水域に到達予定」

### 実装コスト: S〜M（小〜中）

| 項目 | 内容 |
|------|------|
| バックエンド | 忘却確率の集計ロジック（既存 SRS データから算出可能）、天気メタファー変換 |
| フロントエンド | 天気予報ウィジェット（StatsPage に追加） |
| LINE 連携 | 朝の定時通知（EventBridge + Lambda + LINE Messaging API） |
| インフラ | EventBridge スケジューラー追加 |

### 見解
**おすすめ度: ★★★★☆**
既存の SRS データだけで実装でき、追加データ収集が不要。天気メタファーは直感的で、復習の心理的ハードルを下げる効果が高い。LINE 通知はユーザーとの日次タッチポイントになり、リテンション向上に直結。フロントエンドウィジェットだけなら S コストで先行実装可能。

## 11. 記憶のタイムラプス（Memory Timelapse）

### 概要
学習履歴を時系列アニメーションに圧縮し、知識が「育っていく」様子を可視化。年末や節目に自動生成してシェアできる。

### 仕組み
- GitHub の草（contribution graph）のように日々の学習をヒートマップ化
- デッキごとに色分けし、知識の層が地層のように積み重なるアニメーション
- 1 年分を 30 秒の動画に圧縮
- 節目（100 日達成、1000 枚突破等）に自動生成
- SNS / LINE でシェア可能

### 実装コスト: M〜L（中〜大）

| 項目 | 内容 |
|------|------|
| バックエンド | 学習履歴の日次集計 API（既存データから）、動画生成は不要（Canvas アニメーションで対応） |
| フロントエンド | ヒートマップ + アニメーション（Canvas / CSS Animation）、シェア機能 |
| インフラ | 静的画像生成が必要な場合は Lambda + Puppeteer。Canvas のみなら不要 |

### 見解
**おすすめ度: ★★★☆☆**
バイラル効果が期待できるシェア機能だが、LIFF 内でのリッチアニメーション実装は技術的に挑戦的。まずは静的なヒートマップ（GitHub の草風）から始め、アニメーションは後続で追加するのが現実的。StatsPage の延長として段階的に実装可能。

## 12. 五感カード（Synesthesia Cards）

### 概要
視覚だけでなく音・振動・色を組み合わせ、多感覚的に記憶をエンコードする。共感覚的な記憶の定着を促進。

### 仕組み
- カードごとに AI が「記憶の音」を自動生成（Web Audio API）
- 正解時と不正解時で異なる触覚フィードバック（Vibration API）
- カードの難易度が色相で動的に変化（簡単 = 暖色、難しい = 寒色）
- 復習画面全体が自分の理解度を反映した色彩に変わる

### 実装コスト: M（中）

| 項目 | 内容 |
|------|------|
| バックエンド | 変更なし（全てフロントエンドで完結） |
| フロントエンド | Web Audio API でトーン生成、Vibration API、動的カラーテーマ |
| インフラ | 変更なし |

### 見解
**おすすめ度: ★★★☆☆**
バックエンド変更不要でフロントエンドのみで実装できるのが魅力。ただし音や振動の「心地よさ」のチューニングが難しく、下手をするとうるさいだけの機能になるリスク。色相変化は低コストで効果的なので、まず色だけ先行実装するのが良い。

## 13. 反復の考古学（Repetition Archaeology）

### 概要
過去の「忘れて → 思い出した」サイクルの全履歴を「地層」として可視化。自分の記憶パターンの化石を発掘し、メタ学習戦略を発見する。

### 仕組み
- 各カードの全復習履歴を地層ビジュアルで表示
- 「このカードは 4 回忘れて 5 回目で定着。平均より 2 回多い」
- 似た忘却パターンのカードを自動グルーピング →「あなたの苦手パターン」発見
- AI が「このタイプの情報は○○と結びつけると 2 回少ない反復で定着する傾向」と提案

### 実装コスト: M〜L（中〜大）

| 項目 | 内容 |
|------|------|
| バックエンド | 復習履歴の詳細分析 API、パターンクラスタリング、AI 分析 |
| フロントエンド | 地層ビジュアライゼーション、パターン一覧、AI 提案 UI |
| AI/Bedrock | パターン分析結果からの学習戦略提案 |
| インフラ | 変更なし（既存 reviews データを活用） |

### 見解
**おすすめ度: ★★★★☆**
既存の reviews テーブルデータだけで分析可能なのが強み。記憶分解 AI（アイデア 2）と組み合わせると「なぜ覚えられないか × どういうパターンで忘れるか」の両面から攻められ、非常に強力。地層ビジュアルは複雑だが、まずはテキストベースの分析レポートから始められる。

## 14. 記憶の株式市場（Memory Stock Exchange）

### 概要
カードに「記憶株価」をつけ、ユーザー間で価値が変動するマーケットを構築。学習をゲーム経済で駆動する。

### 仕組み
- 各カードの「株価」= 定着度 × 実用頻度 × 希少性
- 他ユーザーが同じカードを学び始めると需要 ↑ で株価上昇
- 定着させたカードは「配当」としてポイント獲得
- 暴落（多くの人が Lapse）したカードを「底値買い」→ 定着させれば大リターン

### 実装コスト: XL（特大）

| 項目 | 内容 |
|------|------|
| バックエンド | 株価計算エンジン、マーケットシミュレーション、ポイント経済、ユーザー間データ集約 |
| フロントエンド | マーケット UI、ポートフォリオ画面、チャート表示 |
| インフラ | リアルタイム集計基盤、ユーザー間データ共有、プライバシー管理 |

### 見解
**おすすめ度: ★★☆☆☆**
ゲーミフィケーションとしては面白いが、実装コストが非常に高く、ユーザーベースが小さいうちはマーケットが機能しない。忘却カジノ（アイデア 1）の方がソロプレイでも成立するため、まずそちらを実装し、ユーザーが増えた段階でソーシャル要素として検討するのが妥当。

## 15. AI デバッガー復習（Rubber Duck Review）

### 概要
不正解時に「答えを見る」代わりに、AI がソクラテス式問答で段階的ヒントを出し、自力で思い出させる。想起の努力（desirable difficulty）を最大化。

### 仕組み
- 不正解時に「答えを見る」「ヒントをもらう」の選択肢
- AI が段階的にヒント: 文脈 → 関連概念 → 部分的手がかり → 最終ヒント
- ヒントなしで思い出せた = 最高評価、ヒント数に応じてスコア減少
- ヒントの種類（語源 / 視覚 / 関連付け）の効果をパーソナライズ
- LINE チャット形式で対話的に実行

### 実装コスト: M（中）

| 項目 | 内容 |
|------|------|
| バックエンド | ヒント生成 AI プロンプト、段階的ヒントセッション管理、ヒント効果の記録 |
| フロントエンド | ReviewPage にヒントモード UI（チャット風）、段階的ヒント表示 |
| AI/Bedrock | ヒント生成プロンプト設計（カード内容から段階的ヒントを生成） |
| インフラ | 変更なし |

### 見解
**おすすめ度: ★★★★★**
逆 SRS（アイデア 5）と並ぶ最推奨。想起の努力（desirable difficulty）は記憶研究で最も効果が実証されている手法。実装コストが中程度で、既存の ReviewPage への拡張として自然に統合できる。逆 SRS と組み合わせれば「教える × 思い出す」の両面で AI を活用でき、Memoru の AI 機能の核になる。

## 16. 記憶の相続（Memory Inheritance）

### 概要
完全定着したデッキを、学習ノート・コツ・つまずきポイント付きで他の学習者に継承。世代を重ねるごとにメタデータが洗練される「集合知デッキ」。

### 仕組み
- 定着済みカードに学習メモが自動蓄積（タイムカプセル復習の発展系）
- 「記憶の遺産」としてデッキを公開
- 次の学習者はカード + 先人の学習体験を受け取れる
- 「この概念は 92% の継承者が 3 回目で定着。先代のコツ:『○○と似てるけど△△が違う』」
- 世代を重ねるごとに学習メタデータが集合知として洗練

### 実装コスト: L〜XL（大〜特大）

| 項目 | 内容 |
|------|------|
| バックエンド | デッキ共有基盤、メタデータ集約、世代管理、統計集計 |
| フロントエンド | デッキマーケットプレイス UI、継承メタデータ表示、先人のコツ表示 |
| インフラ | ユーザー間データ共有基盤、コンテンツモデレーション |

### 見解
**おすすめ度: ★★★☆☆**
「集合知で進化するデッキ」というコンセプトは非常に魅力的だが、デッキ共有基盤・モデレーション・プライバシーの課題が大きい。まずタイムカプセル復習（アイデア 7）で個人のメモ機能を実装し、それを将来的に共有可能にする段階的アプローチが現実的。
