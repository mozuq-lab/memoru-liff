## Dr. ミナト（UX心理学者）の提案

### 1. 未来の自分への手紙（Future Self Continuity Letter）

- **心理学的根拠**: 未来の自分との連続性理論 (Hershfield, 2011) + 実装意図 (Gollwitzer, 1999)
- **機能概要**: 学習の節目（デッキ作成時、ストリーク達成時など）に「1週間後/1ヶ月後の自分」へ短いメッセージを記録。設定した期日に LINE 通知として届き、当時の学習進捗（カード習得率、ストリーク数）と並べて表示される。過去の自分からの励ましと実際の成長データの対比が強力な内発的動機づけを生む。
- **既存技術との相性**: LINE 通知基盤、統計データ（ストリーク、習得率）、DynamoDB、スケジュール Lambda（jobs/）をそのまま活用
- **実装難易度**: 低

### 2. 認知負荷アダプター（Cognitive Load Adapter）

- **心理学的根拠**: 認知負荷理論 (Sweller, 1988) + フロー理論 (Csikszentmihalyi, 1990) + 漸進的開示
- **機能概要**: 復習セッション中のユーザー応答パターン（回答速度、グレード分布、連続不正解数）をリアルタイム分析し、認知負荷レベルを推定。高すぎればヒント付与や選択肢表示に切替、低すぎれば逆方向出題や関連カード混在で負荷を上げる。セッション終了時に「今日のフロー度」を可視化。
- **既存技術との相性**: SM-2 の grade と ease_factor、AI チューター基盤、復習フロー
- **実装難易度**: 中

### 3. 精緻化プロンプト（Elaborative Interrogation Prompt）

- **心理学的根拠**: 精緻化質問法 (Pressley et al., 1987; Dunlosky et al., 2013) + 生成効果
- **機能概要**: 正答後（ease_factor が高い＝簡単と感じているカード）に「なぜこの答えになるのか、30秒で説明してみてください」を表示。AI が説明の妥当性をフィードバックし、不足視点を補足。精緻化履歴はカードに紐付けて保存され、次回復習時に「前回のあなたの説明」として表示。
- **既存技術との相性**: AI チューター基盤、SpeechButton、カードモデル（DynamoDB スキーマレス）、SM-2 の ease_factor
- **実装難易度**: 中

## KIRA（ゲームデザイナー）の提案

### 1. 記憶の討伐クエスト（Memory Raid Boss）

- **ゲームデザイン的根拠**: 期間限定レイドボス（FOMO + 累積進捗可視化 + 適度な不確実性）
- **機能概要**: 毎週月曜にボスが出現。特定テーマ（タグ/デッキ）の問題群で構成され HP を持つ。正答するたびにダメージを与え、grade × 苦手倍率でダメージ計算。日曜までに HP をゼロにすると討伐報酬（称号、ストリークボーナス倍率、限定バッジ）。
- **コアループ**: [月曜] ボス出現 → [毎日] 復習+ダメージ蓄積 → [日曜] 討伐成功で報酬 or 失敗でリベンジ動機
- **実装難易度**: 中

### 2. 記憶コンボチェイン（Recall Combo Chain）

- **ゲームデザイン的根拠**: コンボシステム + 音ゲー型リズムメカニクス
- **機能概要**: 復習セッション中に grade 3 以上が連続するとコンボが積み上がり、ボーナスポイント加算（5 コンボ 1.2 倍、10 コンボ 1.5 倍、20 コンボ 2.0 倍）。視覚エフェクト付き。累積ポイントで「記憶力ランク」（ブロンズ→プラチナ→ダイヤ）が昇格。
- **コアループ**: 正答連続 → コンボ上昇+エフェクト → 途切れ→悔しさ→もう1セッション → ランクアップ
- **実装難易度**: 低

### 3. デイリーミッションボード（Daily Mission Board）

- **ゲームデザイン的根拠**: デイリークエスト + バトルパス型段階報酬
- **機能概要**: AI が弱点データを分析して毎日3ミッション自動生成（例:「苦手カード3枚復習」「新規カード2枚学習」「英語タグ5枚連続正答」）。各達成で星1つ、3星で「パーフェクトデイ」ボーナス。週5日達成でウィークリー報酬。プレイヤーはゲームを楽しみつつ最適な学習をさせられている。
- **コアループ**: [毎朝] ミッション通知 → 達成→星獲得 → パーフェクトデイ → ウィークリー報酬
- **実装難易度**: 中

## Prof. 柊（教育工学者）の提案

### 1. 精緻化プロンプト（Elaborative Interrogation Scaffold）

- **学術的根拠**: 精緻化質問 (Dunlosky et al., 2013) + 処理水準説 (Craik & Lockhart, 1972)
- **機能概要**: grade 3-4 の評価時に「なぜこの答えになるのか」という精緻化プロンプトを自動挿入。AI がフィードバックし、review_history に精緻化回答の有無をフラグ記録。精緻化有無別の保持率変化を統計ダッシュボードで追跡可能に。
- **期待される学習効果**: 深い処理の促進、既存知識との結合、「わかったつもり」の検出・修正
- **実装難易度**: 中

### 2. 文脈交替復習（Contextual Variation Review）

- **学術的根拠**: 文脈変動効果 (Smith & Handy, 2014) + 望ましい困難 (Bjork, 1994)
- **機能概要**: 同一カードの復習時に AI が出題の「提示文脈」を毎回変化させる。1回目は標準、2回目は「友人の誤解を訂正する形式」、3回目は「実際の使用場面を挙げる形式」など。repetitions 数に応じて段階的にバリエーション増加。カードのドメインに応じて調整。
- **期待される学習効果**: 文脈依存記憶からの脱却、「見覚えで答える」浅い再認の防止、長期保持の向上
- **実装難易度**: 中

### 3. 判断較正トレーナー（Calibration Trainer — JOL Feedback Loop）

- **学術的根拠**: Judgment of Learning (Koriat, 1997; Nelson & Dunlosky, 1991) + 自己調整学習 (Zimmerman, 2002)
- **機能概要**: 復習時、答えを確認する「前」に「正解できる自信は？」を3段階で回答。実際の正誤と照合し、過信/過小評価パターンを蓄積。較正スコア（Brier Score 簡略版）を統計ダッシュボードに表示。週次で精度改善のフィードバックを LINE 通知。
- **期待される学習効果**: メタ認知的モニタリング精度の向上、過信バイアス低減、「脆い知識」の特定
- **実装難易度**: 低〜中

## ユキ（LINE エコシステム専門家）の提案

### 1. リッチメニュー学習ダッシュボード（Rich Menu Learning Hub）

- **活用 LINE API**: Rich Menu API (per-user 動的切替) + LIFF URL + Messaging API
- **機能概要**: ユーザーごとに学習状態に応じた Rich Menu を動的切替。未復習カード数、ストリーク、次回復習予定をリッチメニュー画像に直接レンダリング。6分割エリアで「復習開始」「カード追加」「統計」「AI質問」「設定」「デッキ選択」にワンタップアクセス。
- **LINE ならではのポイント**: トーク画面に常駐する UI。アプリを開くたびに学習状態が目に入る。per-user Rich Menu は LINE 公式アカウント独自機能。
- **実装難易度**: 中

### 2. グループ学習バトル（Group Quiz Arena）

- **活用 LINE API**: Group Event + Flex Message + Postback Action + liff.shareTargetPicker()
- **機能概要**: LINE グループにボットを招待し、グループメンバー間で暗記カードを使ったリアルタイムクイズバトル。shareTargetPicker() でグループにクイズ送信、Flex Message で出題、Postback で早押し回答。出題者の SRS スケジュールにも「復習済み」として反映。
- **LINE ならではのポイント**: 既存のソーシャルグラフをそのまま学習コミュニティとして活用。日常会話の延長に学習が組み込まれる。
- **実装難易度**: 高

### 3. トーク履歴からのカード自動抽出（Chat Harvest）

- **活用 LINE API**: LIFF liff.getContext() + liff.sendMessages() + Postback + Flex Message Carousel
- **機能概要**: ユーザーが Memoru に送ったメッセージ群から AI がカード候補を自動抽出。Flex Message Carousel で表示し、Postback で「保存」「編集」「破棄」を選択。既存の generate_cards_from_chunks() を流用。
- **LINE ならではのポイント**: LINE トークの「走り書きメモ」がそのまま暗記カードに。ツール間移動コストをゼロに。
- **実装難易度**: 中

## Neon（SF プロトタイパー）の提案

### 1. 記憶の外部装置化（Exocortex Sync）

- **SF 的着想元**: 攻殻機動隊 — 外部記憶装置、電脳の「外付けメモリ」
- **機能概要**: SRS データから「記憶プロファイル」を自動生成。LINE で「今フランス語の過去形どのくらい覚えてる？」と聞くと、忘却曲線の現在地を推定して「72% 定着。あと2日放置すると危険水域」と返す。外部記憶装置としての脳の状態モニタリング。
- **技術への落とし込み**: AgentCore Memory SDK のセッションコンテキスト + Bedrock Claude の記憶状態サマライザー + LINE Webhook 経由の自然言語クエリ
- **実装難易度**: 低〜中

### 2. 記憶の感染（Mnemonic Contagion）

- **SF 的着想元**: ニューロマンサー + ブラックミラー「White Christmas」のクッキー概念
- **機能概要**: 同ジャンル学習中のユーザー群で、特に苦戦/高速習得したカードのパターンを匿名集計し AI が分析。「このカード、他の学習者の73%がつまずいた。先に免疫をつけておくか？」と Flex Message でプッシュ。記憶の集合知を「感染」メタファーで提供。
- **技術への落とし込み**: 日次バッチ Lambda で匿名集計 + Bedrock Claude で分析 + Flex Message 配信。GSI 1つ追加でジャンル別集計高速化。
- **実装難易度**: 中

### 3. 記憶の時間逆行（Tenet Protocol）

- **SF 的着想元**: 映画 TENET + PSYCHO-PASS のシビュラシステム
- **機能概要**: LINE 利用時間帯、過去の学習完了率、曜日ごとの傾向を Claude が学習し、「明日は水曜だから完了率が40%に落ちる。今日のうちに3枚前倒しでやっておけ」と逆算スケジュールを提案。未来の怠惰を織り込んだプロアクティブ SRS。
- **技術への落とし込み**: 曜日・時間帯別完了率の統計集計 + EventBridge + LINE Messaging API。予測は単純統計で十分。
- **実装難易度**: 中

## クロス分析

### 共鳴ポイント（複数 Agent が独立に類似方向を提案）

1. **精緻化・深い処理** (Dr.ミナト × Prof.柊): 正答後に「なぜ？」を問う精緻化プロンプト
2. **メタ認知の強化** (Prof.柊 × Neon): 判断較正トレーナー + Exocortex Sync（自己認識の可視化）
3. **毎日開く仕組み** (KIRA × ユキ): デイリーミッション + リッチメニュー常駐

### 実装優先度マトリクス

```
            低コスト ←────────────→ 高コスト
  高    │ ★コンボチェイン    │  デイリーミッション │
  効    │ ★較正トレーナー    │  レイドボス        │
  果    │  未来への手紙      │  文脈交替復習      │
        │──────────────────────────────────────│
  中    │  Exocortex Sync   │  記憶の感染        │
  効    │                    │  Chat Harvest      │
  果    │                    │  Rich Menu Hub     │
        │──────────────────────────────────────│
        │                    │  グループバトル     │
        │                    │  Tenet Protocol    │
```

### 最優先候補（低コスト × 高効果）

1. **記憶コンボチェイン** — 復習体験の即時改善、インフラ変更ゼロ
2. **判断較正トレーナー** — reviews テーブルに1フィールド追加でメタ認知を鍛える
3. **未来の自分への手紙** — DynamoDB + LINE 通知だけで動機づけを強化
