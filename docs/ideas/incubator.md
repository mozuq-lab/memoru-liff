# Memoru LIFF 機能インキュベータ

作成日: 2026-03-07

## 目的

まだ `backlog` に載せるには広すぎるが、将来の差別化候補として保持したい機能テーマを整理する。

この文書は、複数のブレスト原文から重複する方向性をまとめ直した「中間整理メモ」である。詳細な発想過程や個別の原文は `docs/ideas/archive/` を参照する。

---

## 1. AI が教える・問い返す学習体験

### 代表案

- ~~AI チューター~~ → **実装済み**（TutorPage: フリートーク / クイズ / 弱点克服の 3 モード）
- 逆 SRS / Teach to Learn
- Rubber Duck Review
- 3秒説明チャレンジ
- 記憶の相棒（AI Buddy） — AI が「クラスメート」として並走し、一緒に悩んだり励ましたりする（`wild.md` より昇格）

### 価値

- 暗記だけでなく「説明できるか」を測れる
- LINE / チャット UI と相性が良い
- 既存の Bedrock 活用と自然につながる

### コメント

AI チューターの基本形は実装済み。今後は「逆 SRS / Teach to Learn」「3秒説明チャレンジ」など、ユーザーがアウトプットする方向の拡張が差別化の次の一手になる。

## 2. PDF / ノートなど入力源の拡張

### 代表案

- ~~URL からのカード生成~~ → **実装済み**（GeneratePage URL モード: カードタイプ / 枚数 / 難易度 / ブラウザプロファイル対応）
- PDF からのカード生成 → `backlog.md` に昇格済み
- AI 要約ノート自動生成
- LINE 会話からのワンタップ追加

### 価値

- カード作成の摩擦を下げる
- 既存の AI カード生成機能を入口から拡張できる
- 「覚える前の整理」と「覚えるための出題」をつなげられる

### コメント

URL 入力は実装済み、PDF はバックログに昇格。残る「AI 要約ノート」「LINE 会話からのワンタップ追加」が次の候補。特に要約ノートは将来の `spec` 候補として扱いやすい。

## 3. メタ認知・弱点分析

### 代表案

- 忘却カジノ
- 自信ギャップ可視化
- つまずきメモ自動収集
- 混同検知モード
- 忘却速度ランキング

### 価値

- 「覚えていない」だけでなく「なぜ覚えられないか」を扱える
- 既存の review データを再利用しやすい
- 学習効率と継続率の両方に効く

### コメント

実装コストに対して価値が高いテーマ群。大きな新規 UI や外部連携なしで試しやすい。

## 4. 継続支援・通知最適化

### 代表案

- 復習ストリーク＆休息バランサー
- Smart Nudge
- スキマ時間フィット
- 予定連動 SRS
- 学習リズム発見
- ニュース連動型・記憶術（News-Link Mnemonics） — 学習キーワードがニュースに登場した際に LINE で通知し、エピソード記憶と結びつける（`wild.md` より昇格）

### 価値

- 継続しづらい SRS アプリの弱点に直接効く
- LINE 通知との相性が良い
- 比較的小さな MVP で検証しやすい

### コメント

プロダクトの継続率改善に直結する。派手さは弱いが、事業的には重要度が高い。

### 代表案

- Card Trading
- Deck Gifting
- 共鳴 / Resonance
- 会話定着ボーナス
- 日常への擬態（Chat Mimicry） — 学習中の単語を使った AI スタンプやサジェスト文章で、日常会話にアウトプット機会を埋め込む（`wild.md` より昇格）

## 7. マルチモーダル・コンテンツ表現の拡張

### 代表案

- カード画像対応 → `backlog.md` に昇格済み
- 音声カード
- Walk & Learn
- 五感カード
