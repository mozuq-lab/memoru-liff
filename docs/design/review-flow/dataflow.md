## エラーハンドリングフロー 🟡

**信頼性**: 🟡 *既存実装パターンから妥当な推測*

```
┌─────────────────────────────────────────────────┐
│                エラーハンドリング                   │
└─────────────────────────────────────────────────┘

 エラー発生
     │
     ├── GET /cards/due 失敗
     │   └── エラーメッセージ + リトライボタン表示
     │       └── リトライ → GET /cards/due 再実行
     │
     ├── POST /reviews/{id} 失敗
     │   └── エラーメッセージ表示
     │       └── 同じ採点ボタンで再送信可能
     │
     └── ネットワーク切断
         └── 採点済みカードは保存済み（API送信成功分）
             └── 未採点カードは復習対象のまま残る
```

## データフロー概要 🔵

**信頼性**: 🔵 *既存API仕様・要件定義より*

### 入力データ

| データ | ソース | 型 | 説明 |
|--------|--------|-----|------|
| due_cards | GET /cards/due | DueCard[] | 復習対象カード一覧 |
| total_due_count | GET /cards/due | number | 復習対象カード総数 |

### 出力データ

| データ | 送信先 | 型 | 説明 |
|--------|--------|-----|------|
| grade | POST /reviews/{card_id} | number (0-5) | 採点結果 |

### ローカル状態

| 状態 | 型 | 初期値 | 説明 |
|------|-----|--------|------|
| cards | DueCard[] | [] | 取得した復習対象カード |
| currentIndex | number | 0 | 現在表示中のカードインデックス |
| isFlipped | boolean | false | カードのフリップ状態 |
| isSubmitting | boolean | false | API送信中フラグ |
| reviewedCount | number | 0 | 採点済みカード数 |
| isComplete | boolean | false | 復習完了フラグ |
| isLoading | boolean | true | 初期読み込み中 |
| error | string \| null | null | エラーメッセージ |
